# AGENTS.md

> このプロジェクトでコードを書く**すべての AI エージェント（Codex / Cursor / Claude Code など）共通の作業規約**です。
> エージェントは作業を始める前にこのファイルを読んでください。Claude Code は `CLAUDE.md` 経由でこの内容を読み込みます。

## ドキュメントの役割

- **README.md** … 人間向け。概要・主な機能・**フォルダ構成**・使い方。**AI 専用の手順（検証ループ等）は書かない。**
- **AGENTS.md**（このファイル）… 全エージェント共通の作業規約・コマンド・**検証ループ**。
- **CLAUDE.md** … Claude Code 固有の補足のみ（このファイルを読み込む薄いラッパ）。
- **graph_mage-spec.md** … 機能仕様と受け入れ条件（実装の合格ライン）。

作業前に：概要とフォルダ構成は README.md、実装する機能と受け入れ条件は graph_mage-spec.md を読むこと。

## Tech Stack

| レイヤー | 技術 |
|---|---|
| Frontend | React 18 + TypeScript + Vite |
| 描画 | HTML5 Canvas（2D context） |
| 数式パース | mathjs（自由入力の式評価） |
| Backend / DB | なし（完全ローカル・サーバ通信なし） |
| テスト | Vitest |
| Lint / 型 | ESLint + TypeScript（`tsc --noEmit`） |
| Deployment | ローカル / 静的ホスティング（GitHub Pages 等） |

## Commands

```bash
npm install        # 依存インストール
npm run dev        # 開発サーバ起動（http://localhost:5173）
npm test           # テスト（Vitest）
npm run lint       # ESLint
npm run typecheck  # tsc --noEmit（型チェック）
npm run build      # 本番ビルド（vite build）
```

> `package.json` の scripts に上記がない場合は、最初のセットアップで追加すること。`typecheck` は `"typecheck": "tsc --noEmit"` を定義する。

## Verification Loop（検証ループ）

**機能を実装したら、完了宣言の前に必ずこのループを回すこと。** これがこのプロジェクトで最も重要な規約。

1. 実装する
2. 検証する：
   ```bash
   npm test         # 受け入れ条件に対応するテスト
   npm run lint     # 静的解析
   npm run typecheck # 型チェック
   npm run build    # ビルドが通るか
   ```
3. graph_mage-spec.md の該当機能の**受け入れ条件**を1つずつ照合する
4. 1つでも失敗・未達があれば、**原因を直して 2 に戻る**
5. すべて green かつ全受け入れ条件 ○ になって初めて、その機能を「完了」とする

### ルール

- テストが落ちている／Lint・型・ビルドにエラーがある状態で「完了しました」と言わない
- 受け入れ条件を満たせない理由が仕様の不備にあるときは、勝手に進めず spec.md の「未決定事項」に追記して相談する
- 描画・アニメーション・体感系の受け入れ条件は自動テストしにくい。**ゲームロジック（`src/game/`）は純粋関数として切り出し、まずそこをユニットテストで固める。** 描画や操作感は手動確認とし、何をどう確認したかを一言報告する
- テストが書ける受け入れ条件は、可能なら先にテストを書いてから実装する（テストが合格ラインになる）

## コーディング規約

- コンポーネントは関数コンポーネント + Hooks のみ。クラスコンポーネントは使わない
- `any` 型は禁止。不明な場合は `unknown` を使い、適切に絞り込む
- **ゲームロジック（関数評価・属性・当たり判定・ダメージ・勝敗）は副作用のない純粋関数として `src/game/` に置く**。React state や Canvas に依存させない
- 自由入力の式評価は mathjs に限定し、`eval` / `new Function` は使わない（安全性のため）
- マジックナンバー（基礎ダメージ・相性倍率・属性しきい値など）は `src/data/` または定数モジュールに集約し、ハードコードを散らさない
- コメントは日本語で簡潔に。1ファイル200行以内を目安にする

## Do NOT

- 新しい依存を勝手に追加しない（mathjs 以外を足すときは相談すること）
- サーバ通信・外部 API・データベースを導入しない（完全ローカル動作が前提）
- `localStorage` などの永続化は MVP では使わない（Phase 2 で検討）
- `eval` / `new Function` で式を評価しない（mathjs を使う）
- README.md に AI 向けの手順（検証ループ等）を書かない（README は人間向けに保つ）

## Notes

- 座標系：Canvas のピクセル座標と、ゲーム内の数学座標（x, y）は別物。**変換関数を1か所にまとめ**、描画・当たり判定の両方でそれを使う（ズレ防止）
- 1プレイ約15分・3ステージ程度を目安にバランス調整する。テンポを損なわないよう、1回の発射〜結果表示は数秒以内に収める
