## 1. 概要

- **一言コンセプト**: 関数を「描いて」魔法に変える、ブラウザで動く約15分のライト RPG。失われた古代遺跡を舞台に、敵の関数を読んでから自分の関数で対抗する、ターン制の関数バトル（8ビット・ドット絵調）。
- **ターゲットユーザー**: 数式に苦手意識のある人を含む、カジュアルに遊びたい一般プレイヤー（自分・友人向け想定）。

> 目的・背景の詳しい説明は README.md を参照（ここでは繰り返さない）。

### コアな考え方（用語の定義）

- **原点 O** … 術者（プレイヤー）の位置を数学上の原点 `O=(0,0)` とする。軌道・防御・全方向発射はこの原点を起点に描かれる。
- **軌道関数** … 弾を飛ばす曲線。2方式：①直交 `y=g(x)` を狙う角度θへ回転、②極座標 `r=f(θ)`。どちらも原点 O 起点。
- **属性の場（z-field）** … `z = f(x, y)` は軌道とは別にプレイヤーが入力する「場」。弾が通る各点 (x,y) で評価し、**符号が属性タイプ、強度（下記）**を決める。場は z 場を編集している間だけ薄く色分け表示する（§3.12）。
- **属性タイプ** … 光（z>0）／闇（z<0）／中立（|z|<ε）。手動選択ではなく、当てる位置の z で決まる。
- **属性強度** … `|z|=zPeak` で最大の**山型**（`strengthOf`）。`|z|=0` と `|z|=2·zPeak` で 0、`|z|=zPeak` で `Smax`。「絶対値が大きいほど強い」ではなく「狙い値 zPeak に近いほど強い」精度ゲーム（#21）。
- **加速度場** … 加速度 `a = aMax(1 − |z|/zRef)`。**`|z|≈0` で最大、`|z|=zRef` で 0、`|z|>zRef` で負（減速）**に転じ `−aDecelMax` で頭打ち。`zRef(=2.5)` は強度ピーク `zPeak(=5)` より手前にあり、最強属性を狙うほど減速する（#31）。
- **運動量／速度** … 弾は固定初速から運動量を保持して進み、加速度場で速度が変化する。障害物・パリィ・周回結界で速度を失い、**速度が 0 になると霧散（消滅）**する。軌道型（周回）も同じく速度0で霧散する。
- **威力** … `命中点の速度 × 属性強度`。速度は飛行中に変化する動的な値。魔法の見た目サイズも威力に連動する（#21/#38）。
- **変異** … 関数の変化量（振れ・曲がり）。大きいほど経路や `|z|` を作り込みやすく、威力に効く。
- **暴発** … 軌道がその地点で未定義・発散・場外になったとき、そこで小範囲ダメージを起こす挙動。
- **周回軌道（結界）** … 攻防は分けず、閉じた曲線（円など）を撃つと「軌道型（周回）魔法」になり防御・支援を兼ねる（§3.11）。敵弾の速度を削って迎撃し、内側の味方を回復／隠蔽する。**破壊されるまでターンをまたいで残る**。旧「シールド」はこれに統合（#39）。
- **障害物（壁）** … 属性／無属性の静的オブジェクト。弾が当たった点を中心に円でえぐり取り、滑らかな穴が開く（Graph War 風・耐久値なし）。当たるたび速度を失い、削り切れず速度0で止まる。壁は4種（属性付き／無属性の壊れやすい・壊れにくい／壊れない）で削れやすさが異なる（§3.7・#40）。
- **パリィ（クラッシュ）** … プレイヤー弾と敵弾が交差したときの解決。同極・中立はすり抜け、反対極（光×闇）のみ速度を削り合って相殺し、速度 0 側が消滅する。
- **ターン進行** … 各ターン「**敵が先に関数を公開 → プレイヤーが見てから関数を作成 → 両者同時発射 → 解決**」を繰り返す。

## 2. 機能一覧

### ✅ MVP（最初に実装する）

各機能には**受け入れ条件**（○× を付けられる条件）を必ず添える。これが検証ループの合格ライン。

#### 機能1：関数の選択・調整 UI（プリセット＋スライダー＋自由入力＋速度＋発射方向＋発射方式）
受け入れ条件：
- [ ] プリセットを §3.10 のカタログから選べる（直交5種：直線・放物線・サイン・指数・絶対値／極座標4種：円・らせん・バラ曲線・リマソン）。選ぶとプレビュー曲線が更新される
- [ ] 係数スライダーは §3.10 の初期レンジで動き、プレビューが即時更新される
- [ ] 自由入力式（`x` の式）を「適用」で mathjs パース＆反映。不正式は「式が正しくありません」を表示し直前の有効関数を維持
- [ ] 発射方式（回転 `y=g(x)` / 極座標 `r=f(θ)`）と、回転時の角度θ、属性の z 場 `z=f(x,y)`（軌道と別入力）を設定できる（初速は固定 `fixedSpeed`・スライダー廃止・#5）
- [ ] 敵が公開した関数（ゴースト）を見ながら自分の関数を編集できる
- [ ] 各プリセットに §3.10 の「説明」が一言表示される

#### 機能2：計算結果の一部表示（数学補助）
受け入れ条件：
- [ ] 軌道関数のサンプル出力（例：`f(0)`, `f(2)`, `f(5)`）が数値表示され、変更で即時更新される
- [ ] 命中見込み点の属性（z>0=光 / z<0=闇 / z≈0=中立）と `|z|`（強度）が表示される
- [ ] 初速・加速の効きやすさ（中立帯を通るか）から、見込み威力の目安が示される（動的なので「目安」）

#### 機能3：2D バトルフィールド描画（原点＝術者・全方向・場の可視化・障害物・敵ゴースト）
受け入れ条件：
- [ ] 術者は原点 `O=(0,0)`、敵・障害物は O 相対座標で全方向（360°）に配置・描画される
- [ ] 属性の場 `z=f(x,y)` は z 場を編集している間だけ薄い色分け（光側／闇側／中立）で表示される（敵・壁より下層・#37）
- [ ] 自分の軌道と、敵が公開した軌道（ゴースト）がオーバーレイ描画される
- [ ] ゲーム座標（原点＝術者）と Canvas ピクセルの変換が1か所に集約され、描画・当たり判定・物理・暴発で共有される

#### 機能4：属性システム（光/闇の極性・z符号＝タイプ・|z|＝強度）
受け入れ条件：
- [ ] `z=f(x,y)` を評価し、符号で属性（光/闇/中立）、`|z|` で強度を返す純粋関数がある
- [ ] 極性相性：反対極（光↔闇）×1.5、同極 ×0.5、中立 ×1.0 が相性表として定義される
- [ ] 状態異常：光＝ひるみ（行動阻害）、闇＝継続ダメージ（DoT）。`|z|` でスケール（§3 参照）

#### 機能5：物理（加速度場・運動量・速度減衰・消滅）
受け入れ条件：
- [ ] 弾は初速から運動量を保持して軌道を進む（数値積分で実装、純粋関数としてテスト可能）
- [ ] 加速度 `a = aMax × (1 − |z|/zRef)`（`−aDecelMax` で頭打ち）を採用し、`|z|≈0` で最大・`|z|=zRef` で 0・`|z|>zRef` で減速（負）になる（#31）
- [ ] 軌道上で速度が積分更新され（弧長エネルギー積分・`maxFlightSpeed` でクランプ）、中立帯で加速・強属性帯（`|z|>zRef`）で減速する（テストで確認）
- [ ] 障害物の削り・パリィ・周回結界の迎撃で速度が減る
- [ ] 速度が 0 になった弾・周回は霧散し、命中しない
- [ ] 命中点の「その時点の速度」を取得でき、威力計算に使われる

#### 機能6：ターン進行と同時発射（敵先出し → プレイヤー後出し → 同時）
受け入れ条件：
- [ ] 各ターン、敵が先に関数（軌道・初速・方式/角度）を決定し、プレイヤーへ公開する
- [ ] プレイヤーは敵の関数を見た状態で自分の関数（攻撃 or 周回結界）を作成・確定できる
- [ ] 「発射」で両者の魔法が**同時**に進行・解決する
- [ ] 解決順序（物理 → パリィ → 障害物 → 命中 → ダメージ/状態異常 → 勝敗）が決まった順で処理される（純粋関数でテスト）
- [ ] 解決後、次ターンへ移行する

#### 機能7：当たり判定（敵・障害物）
受け入れ条件：
- [ ] 飛行中の弾が敵・障害物のヒットボックスを通過したか判定する純粋関数がある（回転/極座標どちらでも一貫）
- [ ] どの敵にも当たらず暴発もしなければ「外れ」、ダメージ 0
- [ ] 速度 0 で消滅した弾は命中しない

#### 機能8：暴発（光と闇を最大強度で併せ持つ特殊術式）
受け入れ条件：
- [ ] 軌道を原点から外側へ評価し、最初に未定義・発散・場外（NaN／±∞／非実数／場外）になった点を暴発点として検出する純粋関数がある
- [ ] 暴発点で小範囲（AoE）ダメージが発生し、範囲内の敵に当たる
- [ ] 暴発は光・闇の両属性を `Smax`（最大強度）で発動し、対象には常に有利側（×1.5）が適用される
- [ ] 暴発はひるみ（光）と継続ダメージ（闇）の両方を付与する
- [ ] `1/x` など原点近傍で即発散する式は術者の近くで暴発し、術者自身を巻き込む自爆リスクがある
- [ ] 暴発でクラッシュせず戦闘継続。見た目は光と闇が混じったデザイン（手動確認可）

#### 機能9：威力・ダメージ計算（動的速度 × 強度 × 相性 ＋ 状態異常）
受け入れ条件：
- [ ] 威力 = 命中点の速度 × 属性強度(`|z|`) で算出される（純粋関数）
- [ ] 最終ダメージ = 威力 × 極性相性倍率（反対1.5／同極0.5／中立1.0）
- [ ] 速度が落ちた状態で当たると威力が下がる（テストで確認）
- [ ] 命中時に状態異常（光＝ひるみ／闇＝DoT）を付与、`|z|` でスケール
- [ ] ダメージ内訳（速度／強度／相性／状態異常）が戦闘ログに表示される

#### 機能10：防御（周回軌道＝結界・#39。§3.11）
受け入れ条件：
- [ ] 閉じた曲線（円など）を撃つと軌道型（周回）魔法になり、攻防を兼ねる（独立シールドは廃止）
- [ ] 周回境界を横切った敵弾の速度を削って迎撃する（反対極のみ相殺・同極/中立は透過）。止めきれば存続、突破されれば霧散
- [ ] 周回の属性は軌道上の符号付き強度の平均で決まり、効果は固定（光=毎ターン+30回復・重複可／闇=隠蔽）
- [ ] 闇の隠蔽は囲む円の半径に連動（1重=半分・2重=完全不可視）。プレイヤー視点は内側を暗化＋ぼかし、敵は参照座標に RMSE の乱数
- [ ] 入れ子は内側（半径小）2つだけ効く
- [ ] 周回は破壊されるまでターンをまたいで残り、反対極の発射魔法に相殺されると消滅する

#### 機能11：障害物（壁・Graph War 風えぐり取り＋壁4種・#1/#16/#40。§3.7）
受け入れ条件：
- [ ] 壁（solids のブロブ）が弾道を遮り、当たった点を中心に円でえぐり取られ滑らかな穴が開く（耐久値なし）
- [ ] 弾が当たると速度が減り、残速度 > 0 なら穴を抜けて継続、0 で停止・霧散する
- [ ] えぐり半径 = 威力 × 係数（最大半径でキャップ）。反対極ほど安く削れて貫通しやすい（純粋関数でテスト可能）
- [ ] 壁4種（属性付き／無属性の壊れやすい・壊れにくい／壊れない）で削れやすさが変わり、壊れない壁は弾を霧散させる（#40）
- [ ] 壁は種別ごとのテクスチャ・色で見分けられ、属性付きは属性色で判別できる（#40）
- [ ] 関数を調整して避ける／削って経路を開けることができる（手動確認可）

#### 機能12：パリィ（同極すり抜け・反対極のみ相殺）
受け入れ条件：
- [ ] 同時発射されたプレイヤー弾と敵弾が交差したら、交差点の場で両弾の属性を判定する（純粋関数でテスト）
- [ ] 同極同士（光×光 / 闇×闇）、または一方が中立のときは、すり抜けて両方とも速度を失わず継続する
- [ ] 反対極（光×闇）同士のときだけ相殺計算が行われ、互いの威力に応じて速度を削り合う
- [ ] 速度 0 になった側は消滅、両者 0 なら完全相殺、残った側は継続する
- [ ] 勝ち残ったプレイヤー弾はそのまま敵へ命中しうる／敵弾を相殺・撃ち勝てば被弾を防げる

## 3. ルール定義

数値は初期値。バランス調整の対象で、`src/data/` または定数モジュールに集約する（§9 も参照）。

### 3.1 座標系（原点＝術者・全方向）

- 術者は数学原点 `O=(0,0)`。敵・障害物は O 相対座標で全方向に配置。
- 発射方式：回転（`y=g(x)` をθ回転）／極座標（`r=f(θ)`）。
- 座標変換は1か所に集約し、描画・当たり判定・物理・暴発・周回結界・障害物で共有する。

### 3.2 属性（光/闇の極性）と場

- `z=f(x,y)` は軌道とは別にプレイヤーが入力する場。弾の各点で `z>0`→光、`z<0`→闇、`|z|<ε`→中立。
- 強度は `|z|` の単調増加ではなく、**`|z|=zPeak` で最大の山型**（`strengthOf`）：`|z|=0` と `|z|=2·zPeak` で 0、`|z|=zPeak` で `Smax`（#21）。
- プレイヤーは属性を直接選ばず、当てる位置の z で光/闇と強度が決まる。場は z 場の編集中だけ薄く色分け表示する（§3.12）。

| 攻撃属性 \ 対象属性 | 反対極 | 同極 | 中立 |
|---|---|---|---|
| 倍率 | ×1.5（有効） | ×0.5（吸収・耐性） | ×1.0 |

### 3.3 状態異常（強度 `|z|` でスケール）

| 状態異常 | 属性 | 効果 | 初期パラメータ |
|---|---|---|---|
| ひるみ | 光 | 次の敵行動を短く阻害 | 例：1ターン詠唱遅延（`|z|` 比例で確率/長さ） |
| 継続ダメージ | 闇 | 数ターン追加ダメージ | 例：`|z|`×2 を数ターンに分割 |

### 3.4 物理（加速度場・運動量・速度・消滅）

```
属性強度   = Smax × (1 − ||z(p)| − zPeak| / zPeak)   # |z|=zPeak で最大、|z|=0 と 2·zPeak で 0（山型）
加速度     a(p) = clamp(aMax × (1 − |z(p)| / zRef), −aDecelMax, +∞)
                                                     # |z|≈0 で最大、|z|=zRef で 0、|z|>zRef で負（減速）
速度更新   v² ← v² + 2·∫a ds（弧長積分。v ← v + a·dt と等価）、v は maxFlightSpeed でクランプ
威力       = v命中点 × 属性強度命中点
最終ダメージ = 威力 × 極性相性倍率
```

**主要定数（`src/data/constants.ts` の実装値・すべて調整対象）**

| 定数 | 記号 | 値 | 説明 |
|---|---|---|---|
| 場外境界 | `R_field` | 30 | 原点からこの半径外は「場外」＝暴発/消滅 |
| 強度上限 | `Smax` | 5 | 山型強度の最大値 |
| 強度ピーク | `zPeak` | 5 | `|z|=zPeak` で強度が最大（狙い値） |
| 中立しきい | `ε` | 0.35 | `|z| < ε` を中立 |
| 加速度上限 | `aMax` | 13 | `|z|≈0` での加速度（最大） |
| 加速度基準 | `zRef` | 2.5 | `|z|=zRef` で加速度 0。これより 0 から離れると減速（#31） |
| 減速上限 | `aDecelMax` | 8 | `|z|>zRef` の減速の頭打ち |
| 積分ステップ | `dt` | 1/60 | 固定 dt で数値安定 |
| 初速 | `fixedSpeed` | 10（固定） | 毎回固定（スライダー廃止・#5） |
| 終端速度 | `maxFlightSpeed` | 24 | 加速で際限なく伸びるのを防ぐ上限 |
| AoE 半径 | `aoeRadius` | 5 | 暴発の範囲 |

> 設計の含意（#31）：強度ピーク `zPeak(=5)` は加速度0点 `zRef(=2.5)` より外側にある。**最強属性（|z|≈zPeak）を狙うほど弾は減速し、速度0で霧散する**ため近距離限定の大威力になる。一方 `|z|≈zRef` は減速しない代わりに強度が中庸で中遠距離に届く。**威力（強度）と射程（速度維持）のトレードオフ**を突く精度ゲーム。

- 弾は運動量（速度）を保持して進む。場の加速度で速度が増減（弧長エネルギー積分）。
- **速度減衰イベント**：障害物の削り・パリィ・周回結界の迎撃で速度を失う（§3.6/§3.7/§3.8/§3.11）。
- **霧散（消滅）**：速度が 0 になった弾・周回は消える。発射型は終端で**小さくなって霧散する演出**（#38）。
- **核となる駆け引き**：中立帯（`|z|≈0`）は最も加速するが強度ゼロ。`|z|=zRef` は減速せず中遠距離に届く。`|z|=zPeak` は最大威力だが減速して近距離限定。経路で加速し、当てたい距離に応じて差し込む `|z|` を選ぶ。

### 3.5 暴発（光と闇を最大強度で併せ持つ）

- エラー定義：軌道評価が NaN／±∞／非実数／場外（原点から `R_field` 外）。
  - **z 場（属性関数）のエラーも同様に暴発する（#30）**：軌道は有効でも、経路上で z 場が非有限（`sqrt(-1)`・`log(0)`・`1/x` の極など）になる点で暴発する。暴発点はエラー手前の場内有効点。
- 原点から外側へ評価し、**最初にエラーになった点**を暴発点とする。
- **暴発は光と闇の両属性を `Smax`（最大強度）で同時に発動する、常に最大威力の特殊な術式**。デザイン上も光と闇が混じった見た目にする。
  - ダメージ：**`威力 = 常に最大（Smax × maxFlightSpeed）`**（#41：暴発時の速度に依らず一定で最大）。両極性を併せ持つため、対象の属性に対し**常に有利側（×1.5）が適用される**（光・闇どちらかが必ず弱点を突く）。
  - 状態異常：**ひるみ（光）と継続ダメージ（闇）の両方**を `Smax`（最大強度）で付与。
  - 範囲：小半径 AoE（`aoeRadius = 5` ユニット）。範囲内の**生存している**敵・味方すべてに適用（撃破済み hp0 にはダメージ判定を出さない）。
  - **壁の破壊（#41）**：AoE 内の壁（障害物）も**ほぼ全て削れる**（AoE 円ぶんを丸ごとえぐる）。`unbreakable`（壊れない壁）だけは残る。
  - **演出と効果範囲を一致させる（#29）**：暴発アニメーションの最大半径は `aoeRadius` に一致させ、AoE 境界の円を明示する（見た目より広く/狭くダメージが入る、という誤解を防ぐ）。
  - **演出（#41）**：**紫（闇）と黄（光）の腕がぐるぐる回りながら中心へ集まる大渦**（虚式・茈のイメージ）。加算合成で、紫と黄は完全には混ざらず色を残しつつ中心は白熱（最後は白く埋まる）。**効果範囲の内側を渦で埋め尽くす**（少しだけ外へはみ出し、AoE 境界の円は別途明示）。爆発で**ステージ全体が揺れ**、上空から**遺跡の破片**が降ってくる。
- ただし発生地点は制御しづらく、`1/x` 等は原点近傍で即暴発 → **術者自身を巻き込む自爆リスク**。狙って当てれば切り札、事故れば自滅という高リスク・高リターン。

### 3.6 防御（周回軌道＝結界に統合・#39）

- 攻防は分けない。閉じた曲線（円など）を撃つと**軌道型（周回）魔法**になり、防御・支援を兼ねる（詳細は §3.11）。
- 敵弾が周回の境界を横切ると速度を削って迎撃する（反対極のみ相殺・同極/中立は透過）。止めきれば結界の勝ち（存続）、突破されれば霧散。
- 旧「閉曲線シールド」はこの周回軌道に統合（独立したシールド構造は廃止）。

### 3.7 障害物（壁・Graph War 風のえぐり取り＋壁4種・#1/#16/#40）

- 形は `solids`（重なった円の和＝連続したブロブ）。耐久値は持たず、弾が当たった点を中心に円（`carves`）を引き算して**物理的にえぐり取る**（滑らかな穴が開く）。
- 当たるたび速度を失い、残速度 > 0 → 削った穴を抜けて継続、速度0 → 停止・霧散。えぐり半径 = 威力 × 係数（最大半径でキャップ）。反対極ほど安く削れて貫通しやすい。
- **壁4種（種別ごとに削り半径・速度損が変わる・#40）**：

| 種別 | element | 削れやすさ | 用途 |
|---|---|---|---|
| 属性付き（normal） | 光/闇 | 反対極で速く削れる（相性が効く） | 従来の属性壁 |
| 無属性・壊れやすい（fragile） | neutral | 一撃で大きく削れる | もろい囲い・装飾 |
| 無属性・壊れにくい（tough） | neutral | ごく小さくしか削れず、貫通に複数発（目安5発以上） | 補強された隔壁 |
| 壊れない（unbreakable） | — | 素材は削れない。当たった魔法はその場で霧散 | 迂回必須の芯柱・封印核 |

- 壁は種別ごとに異なるテクスチャ（目地／ひび／鋲打ち／鋼の斜線）と色で見分けられ、属性付きは属性色で判別できる（#40）。ステージごとに装飾や味方間の仕切りも配置してよい（方眼の目盛り＝関数値は全ステージ共通スケールで一致）。

### 3.8 パリィ（クラッシュ解決＝同極すり抜け・反対極のみ相殺）

- 同時発射された両弾が交差したら、**交差点の場で両弾の属性を判定**する。
- **同極同士（光×光 / 闇×闇）、または一方が中立** → **すり抜け**。相互作用せず、速度を失わず両方そのまま継続する。
- **反対極同士（光×闇）** → **相殺計算**：双方の速度を相手の威力（`速度 × |z|`）に応じて削り合う。
  - 速度 0 になった側は消滅、残った側は継続。両者が 0 なら完全相殺。
- 戦術的含意：敵の光弾を止めたいなら、交差点で闇になる弾をぶつける（逆もまた然り）。同極では止められない。
- 入射角を補正に加える案は §9 のTODO。

### 3.9 世界観・デザイン方針

**テーマ**：失われた古代文明の遺跡を巡る、魔導士見習いの物語。古代人は世界の理（ことわり）を「数式」で記述し、それが魔法文字＝術式となった。遺跡には《光の理》と《闇の理》が満ち、術者は古代式を読み解いて魔法を放つ。

**ストーリー骨子（3ステージ）**：遺跡の入口 → 地下回廊 → 最奥の守護者（封印）。各ステージで古代式の断片（新しい関数）や仕掛け（障害物・場）が登場し、最後に魔導書の核心へ到達する。短いテキストで導入とクリアをつなぐ。

**用語**：魔法＝術式／関数＝古代式（魔法文字）／場＝理の場／`f>0` 側＝光の理・`f<0` 側＝闇の理／シールド＝結界／暴発＝術式の綻び。

**ビジュアル・トーン**：8ビット・ドット絵風、ノスタルジック。粗いピクセルグリッド、石碑や魔導書を思わせるチャンキーな枠。`image-rendering: pixelated` と低解像度キャンバスの整数倍拡大でピクセル感を出す。任意でスキャンライン。

**配色（高コントラスト・背景黒め）**：
- 背景：ほぼ黒（例 `#0d0b14`）。グリッドは暗い藍。
- 光の理：金 `#F4C430` 〜 白 `#FFF8E1`（暖色寄り）。
- 闇の理：紫 `#7B5CC4` 〜 藍 `#1E2A6B`。
- 場の色分け：`f>0`＝金のグラデ、`f<0`＝紫藍のグラデ、`f≈0`（中立）＝暗いグレー。
- 弾・エフェクト：光＝金のスパーク、闇＝紫の揺らぎ。ドット粒子で表現。

**フォント**：日本語はレトロなドット系（例 `DotGothic16`）、見出しにピクセル系ディスプレイフォント。可読性を優先する。

**実装方針**：色・フォント・間隔はトークン化し、frontend-design スキルの指針に沿って一貫させる（数値は実装時に微調整可）。

### 3.10 関数カタログ（実装する関数）

実装する関数の定義。係数は MVP のスライダー初期レンジを併記（調整可）。各関数の「説明」はチュートリアル／関数解説（機能1・16）でそのまま提示する。

**A. 軌道（直交 `y=g(x)`・狙う角度θで回転）**

| 名称 | 式 | 主な係数 | 説明（チュートリアル文） |
|---|---|---|---|
| 直線 | `y = a·x + b` | a∈[-3,3], b∈[-5,5] | まっすぐ飛ぶ基本の術式。狙いを定めやすい。 |
| 放物線 | `y = a·(x−h)² + k` | a∈[-1,1], h∈[0,10], k∈[-5,5] | 山なり／谷なりの弧。障害物を山越えできる。 |
| サイン波 | `y = A·sin(B·x) + C` | A∈[0,4], B∈[0.2,2], C∈[-3,3] | 波打って飛ぶ。光と闇の場を縫える。変異が大きい。 |
| 指数 | `y = a·e^(b·x) + c` | a∈[0,2], b∈[-1,1], c∈[-3,3] | 終盤で急上昇。遠くで一気に跳ね上がる。 |
| 絶対値 | `y = a·|x−h| + k` | a∈[-2,2], h∈[0,10], k∈[-5,5] | V字に鋭く折れる。きっかけで方向を変える。 |

**B. 軌道（極座標 `r=f(θ)`・全方向）**

| 名称 | 式 | 主な係数 | 説明（チュートリアル文） |
|---|---|---|---|
| 円 | `r = R` | R∈[1,12] | 一定半径で全周をなめる。結界の基本形にも。 |
| らせん | `r = a·θ` | a∈[0.2,2] | 渦巻き状に外へ。全方向を順に薙ぐ。 |
| バラ曲線 | `r = a·cos(k·θ)` | a∈[1,10], k∈{2,3,4,5} | 花びら状に複数方向へ同時に伸びる。変異が大きい。 |
| リマソン | `r = a + b·cos(θ)` | a∈[1,6], b∈[1,6] | ハート／くぼみ形。片側に強く張り出す。 |

**C. 属性の場 `z=f(x,y)`（ステージごとにシステム側で設定）**

| 名称 | 式 | 光/闇/中立の出方 |
|---|---|---|
| 平面傾斜 | `f = a·x + b·y` | 片側が光、反対が闇。中立は一直線の帯。 |
| 同心円 | `f = x²+y² − c` | 中心が闇、外周が光（c で半径）。中立は円環。 |
| 市松（波） | `f = sin(a·x)·sin(b·y)` | 光闇が網目状に交互。中立帯が格子状に走る（加速ルート向き）。 |
| 双曲 | `f = x² − y²` | 4象限で光闇が交互。中立は対角線。 |

**D. 結界（防御の閉曲線）**

| 名称 | 式 | 説明 |
|---|---|---|
| 円結界 | `x² + y² = R²` | 全方向を均等に守る。 |
| 楕円結界 | `x²/a² + y²/b² = 1` | 特定方向に厚く張れる。 |

### 3.11 周回軌道魔法（軌道型＝結界・#39）

閉じた曲線（極座標の円など）を撃つと「軌道型（周回）魔法」になり、攻撃ではなく防御・支援を担う。発射型（火球）とは別系統。

- **属性の決め方**：軌道上の**符号付き強度の平均**で属性（光/闇/中立）を決める。**効果に強度の大小は使わない**（属性＝符号だけが意味を持つ）。
  - これは「相殺」での参照と**異なる**ので注意：発射魔法と相殺するときは「当たった箇所の」属性強度・速度を参照する（下記）。
- **永続化**：一度張った周回は**破壊されるまでターンをまたいで残る**。毎ターン内側へ効果を及ぼし、敵弾を迎撃する。撃ち直さなくてよい。
- **相殺（破壊）**：反対属性の発射魔法が境界を横切ると相殺する（同極・中立は透過）。計算は発射魔法のパリィと同種で、当たった箇所のリング強度×速度と敵弾の速度を参照。リングが止めきれば存続、突破されれば消滅（次ターンへ持ち越さない）。壁に触れても霧散する。
- **内側への効果**（囲んだ味方に作用）：
  - **光**：毎ターン、効果範囲内の味方の体力を **+30 回復**（強度に依らない固定量・**重複可**）。
  - **闇**：味方を隠蔽。**1つの円で視認性が半分**（プレイヤー視点はその円の内側が**暗くなり、さらにぼやける**／敵は参照座標に RMSE＝半径×1/2 の乱数が乗る）。**2つの円で完全に視認不可**（同じ幕を2枚＝ほぼ真っ暗、敵の RMSE＝半径相当）。
  - **入れ子の優先**：複数の円に囲まれた場合、効果は**内側（半径が小さい）の最大2つ**だけが効く。例：光①・闇②・光③（内→外）なら ①②のみ。
  - 1つの関数で2つ以上の円を出した場合：相殺で破壊されたら全て消える。効果の重複・入れ子優先は上記と同じ（実装上は複数リング一般で扱う）。
- **粒・サイズ**：周回の粒の大きさは威力（速度×強度）に連動（#21）。

### 3.12 z 場の表示と演出（#37/#38）

- **z 場の可視化（#37）**：z 場（属性関数）を**いじっている間だけ**、その実行結果をステージに薄い「場」として表示する（敵・壁より下のレイヤ）。編集していないときは表示しない。
- **暴発点の可視化（#30）**：作成フェーズのプレビューで、軌道・z 場のどちらかがエラーで暴発する点を**赤い ✕ マーカー**で表示する（最前面）。場外脱出（ただの外れ）には出さない。撃つ前に「弾がどこで暴発するか」を確認できる。
- **場のエラー地点の可視化（#30）**：z 場を編集している間、アクティブな術者の z 場が**エラーになる地点を全て**赤で重ねる。極（`1/x` 型の発散）は**赤い線**、定義域外（`sqrt(-1)`・`log(0)` など）は**赤い領域**として現れる。場のどこが危険かを一目で把握できる。
- **軌跡 UI の整理（#37）**：前ターンに撃った魔法の軌跡表示は廃止。作成フェーズの「どこで途切れるか（着弾点）」表示も廃止し、これから撃つ魔法の**軌跡のみ**を表示する。
- **ダメージ／回復の数値（#42）**：敵・味方が被弾・回復するたび、対象の頭上に数値が浮かび上がる。**色は属性色**（光=金/闇=紫/中立=淡）、**暴発=白**、**回復=緑（`+`付き）**。**大きさは受けた量に依存**（大ダメージほど大きい）。
- **魔法サイズ（#21/#38）**：発射魔法・周回の粒の見た目サイズは威力に連動する。
- **霧散演出（#38）**：速度0で消える発射魔法は、**小さくなって霧散**する（周回軌道の霧散と同じ質感）。
- **衝突パーティクルと音（#38）**：
  - パリィ時・周回結界に当たったとき＝**青いパーティクル**を混ぜ、「**バチッ**」という音を出す。
  - 壁に当たったとき＝**赤いパーティクル**（発射型・周回型どちらも）。
  - パーティクルの大きさは威力に依存（パリィ時は**2魔法の威力の合計**に依存）。

## 5. データモデル

> 永続化はしない（メモリ上の状態のみ）。

### Attribute / FireMode

- `Attribute = "light" | "dark" | "neutral"`（命中位置の `z` の符号で決定）
- `FireMode = "rotate" | "polar"`

### Trajectory（軌道：プレイヤー／敵共通）

- 軌道は `RotateTrajectory { mode:"rotate", g(x), angle, origin, z? }` または `PolarTrajectory { mode:"polar", f(θ), origin, z? }` の判別共用体。
- `z`（z 場 `(x,y)=>number`）は軌道とは別に持ち、弾が通る各点で属性・強度・加速度を決める（#30）。初速は固定（`fixedSpeed`）。
- 飛行結果は `Flight { samples[], end, endPos, endSpeed }`。`end` は `vanished`（速度0）/`outOfField`/`invalid`（暴発）/`maxParam`。

> 属性・強度・威力は飛行/命中/クラッシュ位置の `z` と現在速度から算出する。

### Obstacle / Enemy / Ally / ActiveOrbit / StatusEffect / Stage

| 構造 | 主なフィールド |
|---|---|
| Obstacle（壁） | `id`, `element`, `solids: Disc[]`（円の和）, `carves: Disc[]`（えぐった穴）, `kind?`("normal"\|"fragile"\|"tough"\|"unbreakable") |
| Enemy | `id`, `pos`, `hp`,`maxHp`, `element`, `hitboxRadius`, `family`/`families`(得意関数), `role?`("attacker"\|"breaker"\|"guardian"), `castTrajectory`, `castInitialSpeed`, `castZ`, `castZField?` |
| Ally | `id`, `pos`(=発射元), `hp`,`maxHp`, `element`, `statuses`, `concealed?`(闇周回の重数), `concealRmse?`(隠蔽のブレ幅) |
| ActiveOrbit（持続周回・#39） | `id`, `ownerId`, `owner`, `ring: ZPoint[]`, `ringSpeed`。破壊まで `BattleState.orbits` に残る |
| StatusEffect | `kind`("flinch"\|"burn"), `magnitude`(|z|由来), `remainingTurns` |
| Stage | `id`, `name`, `enemies`, `obstacles`, `introText`, `clearText`, `mechanics`(段階導入), `boss?` |

> z 場（属性）はステージ固定ではなく、術者ごとに軌道へ紐づけて入力する（#30）。状態異常・持続周回はターンをまたいで状態管理する。
