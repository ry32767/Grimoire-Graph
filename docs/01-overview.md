# 01. 概要・コアコンセプト・世界観

## ゲーム概要

**Graph Mage（グラフメイジ／内部名 Grimoire Graph）** は、関数を「描いて」魔法に変える、ブラウザで完全ローカル動作するターン制の関数バトル RPG。
プレイヤーは **3 人の術者からなる自陣営** を率い、古代遺跡の 7 つの間を巡って敵チームと撃ち合う。1 プレイ約 15 分。

- フロント：React 18 + TypeScript + Vite
- 描画：HTML5 Canvas（2D context）
- 数式パース：mathjs（自由入力式の安全評価。`eval`/`new Function` は不使用）
- バックエンド／DB：なし（サーバ通信なし・永続化なし）
- テスト：Vitest

## コアコンセプト（用語の定義）

### 軌道（経路）と z 場（属性）は別物

このゲームの心臓部は、**「弾がどこを飛ぶか（軌道）」と「その点がどんな属性か（z 場）」を分離している**こと。

| 概念 | 定義 | 入力 |
|---|---|---|
| **軌道 Trajectory** | 弾の通る経路。回転方式 `y=g(x)` か極座標方式 `r=f(θ)`。 | 軌道プリセット or 自由入力式 |
| **z 場 ZField** | 位置の 2 変数関数 `z=f(x,y)`。弾が通る各点 (x,y) で評価し、属性・強度・加速度を決める。 | z 場プリセット or 自由入力式 |

弾は軌道に沿って飛びながら、各点で z 場を評価する。z の符号が属性、|z| が強度、|z| が加速度を決める。

### 属性 = z の符号

```
z > +epsilon  → 光（light）
z < -epsilon  → 闇（dark）
|z| < epsilon → 中立（neutral）   epsilon = 0.35
```

光と闇は反対極。**反対極に当てると ×1.5、同極は ×0.5、中立絡みは ×1.0**（相性倍率）。

### 強度 = |z| の「山型」

「絶対値が大きいほど強い」ではなく、**ある値 V = zPeak(=5) に近いほど強い**山型関数。
|z|=0 と |z|=2·zPeak(=10) で強度 0、|z|=zPeak(=5) で最大 sMax(=5)。

### 速度と消滅 = 中立が最速、強属性は失速

中立（z≈0）で最大加速、|z|=zRef(=2.5) で加速 0、それを超えると減速に転じる。
**最強属性 zPeak(=5) は zRef を超えているため、強く帯びさせるほど弾は減速し、速度 0 で霧散する。**
→「中立帯で加速 → 当てる瞬間に強く帯びさせる」という精度ゲームになる。

> 計算式の詳細は [02-calculations.md](02-calculations.md) を参照。

### 二系統の魔法

- **発射型（projectile）** … 開いた軌道（直線・放物線など）。術者から飛び、最初に当たった対象へ命中。
- **軌道型（orbit）＝結界** … 閉じた軌道（円・バラ曲線など）。術者の周りを回り、敵を掃射しつつ敵弾を迎撃（防御）する。光の結界は回復、闇の結界は隠蔽の効果も持つ。一度張ると破壊されるまでターンをまたいで持続する。

> 詳細は [04-magic.md](04-magic.md) を参照。

## 世界観・トーン（§3.9）

- **テーマ**：失われた古代文明の遺跡を巡る魔導士見習いの物語。数式は古代の魔法文字＝術式。遺跡には《光の理》と《闇の理》が満ちる。
- **トーン**：8 ビット・ドット絵調、ノスタルジック。石碑や魔導書を思わせるチャンキーな枠、ピクセル表現。
- **配色（高コントラスト・背景黒め）**：光の理＝金〜白、闇の理＝紫〜藍、背景はほぼ黒。弾の軌道は z で色分け（正＝金、負＝紫藍、中立＝淡色）。

## 自陣営パーティ（`src/data/party.ts`）

## 画面フロー（`src/App.tsx`）

```
title（タイトル）
  └→ prologue（序章）
       └→ stageIntro（ステージ導入）
            └→ battle（戦闘：作成⇄解決を繰り返す）
                 ├→ stageClear（ステージクリア）→ 次の stageIntro（最終面なら ending）
                 ├→ gameover（全滅）→ やり直し or タイトル
                 └→ ending（エンディング・クリアタイム表示）
```

- **クリアタイム計測**：タイトルの「はじめる」で `runStartMs` を記録し、各ステージクリア時に経過を表示。
- **チュートリアル**：ステージ 1 開始時に遊び方ガイドを自動表示（1 回のみ）。
- **図鑑（Codex）**：遭遇した敵・関数の解説を閲覧（セッション内のみ・永続化しない）。
- **音**：タイトルの「はじめる」で AudioContext を resume し BGM 開始。🔊/🔇 でミュート切替。
- **DEV ステージ直行**：開発ビルドのみ `?stage=N`（1 始まり）やタイトルのボタンで任意ステージへ直行可（本番ビルドでは無効）。

### 戦闘画面の 1 ターン

1. `prepareTurn`：状態異常をターン減衰（継続ダメージ適用・ひるみ判定）。発射できる敵 ID・ひるみ味方 ID を決定。フェーズを `compose` へ。
2. 作成フェーズ：味方タブで切り替えつつ、各自の軌道関数・z 場・狙い角を編集。「おすすめ」ボタンで自動提案も可能。
3. 「全員発射」：`resolveAllyCasts` → `resolveTurn` でまとめて解決。`ResolveAnimation` を組み立てて `BattleCanvas` に渡し、演出を再生。
4. 演出完了（`onAnimationDone`）で勝敗を判定し、次のターンへ（または clear/gameover 画面へ）。

> ターン進行・解決順序の詳細は [09-data-model.md](09-data-model.md) を参照。
