# 05. 敵・敵 AI

敵の定義は `src/data/stages.ts`（`enemy()` ファクトリ・LVL スケール）、AI は `src/game/enemyAI.ts`。

> 攻撃パターン4種（火力型/迂回型/暴発型/守護型）と、パターンごとに使える軌道関数・z場関数の組み合わせは [05b-enemy-archetypes.md](05b-enemy-archetypes.md) にまとめている。本ファイルは `role`/`family` の基礎仕様、05b はその上に乗るパターン設計。LVL ごとの解放・数値は [06b-difficulty-framework.md](06b-difficulty-framework.md)。

## 5.1 敵の構造（`Enemy` 型）

| フィールド | 意味 |
|---|---|
| `name` / `pos` / `hp` / `maxHp` | 名前・配置（=発射元）・HP（LVL 倍率 × 基礎100・[06b](06b-difficulty-framework.md) §2） |
| `element` | 被ダメージ相性に使う**防御属性**（light/dark/neutral） |
| `hitboxRadius` | 被弾ヒットボックス半径（既定 `enemyHitbox=1.8`、ボスは 3.6） |
| `family` | 得意関数の**系統**（見た目で判別・AI が最適化する関数族） |
| `families?` | 追加系統（#28：1〜2 個）。AI は family＋これらを全部試して最良を選ぶ |
| `role?` | 戦い方（attacker / breaker / guardian / ruptor）。未指定は attacker |
| `castInitialSpeed` | 敵弾の初速。**全敵・全LVLで 8 固定**（06b §2） |
| `castZ` | 敵弾の代表 z（符号=属性の基準）。`castMag`（LVL 依存）×極性 |
| `castZField?` | 敵弾の z 場（未指定なら攻撃時に反対極の一定場を選ぶ）。sin/cos 場などに使う |
| `ruptorTarget?` | 崩し手の狙い先。`'obstacles'`＝壁狙い（第4面デモ）。未指定は味方狙い |
| `castCount?` / `patternPool?` | 多重詠唱（#44・05b §5.5）：同時発射数とパターンの許可プール |
| `fireEvery?` / `fireOffset?` | 発射頻度（06b §2）：この間隔（ターン）ごとに発射。`(turn+offset)%every===0` のターンに撃つ |
| `boss?` | ボス個体（#45：HPフェーズ・断末魔の対象） |
| `slipThrough?` | 迂回型高難度：同極すり抜け（05b §5.2） |
| `alternatingAura?` / `guardZSign?` | 守護型高難度：交互張り（05b §5.4）。符号は `prepareTurn` がターン偶奇で設定（奇数=光） |

`enemy()` ファクトリは `castZ = element==='light' ? +castMag : element==='dark' ? −castMag : 0` を設定。
ただし AI は攻撃時、**属性に関係なく対象の弱点（反対極）を自由に突く**（§5.4）ので、`element` は主に被弾相性と見た目に効く。

## 5.2 系統（family・#17/#43）

弾の形で判別できる「得意関数」。頭上の記号（glyph）とゴースト軌道で予告される。

| family | ラベル | AI が組む軌道（`buildEnemyTrajectory`） | 形状係数候補 |
|---|---|---|---|
| `line` | 直進 | `g(x)=0`（直線） | `[0]` |
| `arc` | 弧 | `g(x)=shape·x²`（緩い放物） | `[-0.09, -0.04, 0.04, 0.09]` |
| `wave` | 波 | `g(x)=shape·sin(0.45x)` | `[1.5, 3]` |
| `spiral` | 渦 | `f(t)=shape·(t+angle)`（極座標・渦巻き） | `[0.7, 1.1]` |
| `exp`（#43） | 昇り | `g(x)=shape·(e^(0.13x)−1)`（指数・終盤で鋭く跳ね上がる） | `[-0.8, -0.35, 0.35, 0.8]` |
| `poly34`（#43） | 捻れ | `g(x)=shape·(x³−140x)`（3次・S字/こぶ） | `[-0.004, -0.002, 0.002, 0.004]` |

`ARCHETYPES` がラベルと glyph を保持。狙い角オフセットは spiral 以外で `[-0.28,-0.14,0,0.14,0.28]`、spiral は `[0]`。

> `ruptor` は `line` を除く family を使う（暴発型は迂回型の制約を継承・05b §2）。見分けは family の glyph に加えて、**専用のひび割れ記号（縁の赤い稲妻）**と、破綻点に出る赤い✕＋揺れる円で行う（§5.4b）。

## 5.3 ロール（role・#28/#42）

| role | パターン名（05b） | 説明 |
|---|---|---|
| `attacker`（既定） | 迂回型 | 味方へ最大ダメージを狙う。壁は避けて通る（`avoiderTrajectories`）。 |
| `breaker` | 火力型 | 壁を貫いてでも味方へ届かせる。**障害物ペナルティを受けない**（壁よけ軌道も使わず直進で貫く）。 |
| `guardian` | 守護型 | 攻撃せず、自陣を守る**防御用周回結界**を張る。半径は `enemyGuardRadius=7` を基本に、**壁に触れない半径**（×0.75/×0.55）へ縮めて選ぶ。z は自陣属性（交互張り個体は `guardZSign`）×`zRef` で展開し味方弾を迎撃する。光の結界は囲んだ敵陣を毎ターン `orbitHealAmount(=30)` 回復する（05b §5.4）。 |
| `ruptor`（崩し手・#42） | 暴発型 | 味方（または壁）を狙い、**z 場に極を仕込んで対象近傍で暴発**を起こす（04-magic §4.3 の暴発と同一）。威力は暴発仕様どおり常に最大・光闇両属性。**詳細は §5.4b**。 |

guardian の結界も「反対極のみ相殺・同極は透過・壁で霧散・強属性の失速で自滅（#31/#44）」という味方の結界と同じルール（[04-magic.md](04-magic.md) §4.6）。ただし guardian は減速しない最強（|z|=zRef）で張るので失速自滅はしない。**ruptor の弾も例外ではなく、この同じルールに従う**（§5.4b）。

## 5.4 敵 AI の攻撃計画（`planEnemyShot` / `planEnemyShots`）

最大ダメージの軌道を探索する。多重詠唱（`castCount>1`・#44）は `planEnemyShots` が弾ごとに独立へ分解する（05b §5.5）。流れ：

1. **guardian** なら防御結界を張って終了（`planGuardianOrbit`）。**ruptor** なら専用計画（§5.4b）。
2. **隠蔽**：闇の周回で完全に隠れた味方（`concealed ≥ orbitConcealFull`）は視認不可＝狙えない。全員隠れていれば見えないなりに撃つ。
3. 各「狙う味方 × 狙い角 × 形状係数 × z 場強さ」の候補軌道を総当たりで採点（`consider`）。

### z 場の強さ候補（#31）

```ts
ATTACK_Z_MAGS = [zPeak(=5), zRef(=2.5)]
```

対象の反対極を、`zPeak`（近距離で大威力だが減速）と `zRef`（中遠でも届く）の両方で試す。`castZField` を持つ敵はそれ 1 つを使う。

### 同極すり抜け（高難度迂回型・`slipThrough`・05b §5.2）

狙う味方が**持続結界（前ターンまでの周回）に囲まれている**とき、その結界の平均属性（`ringAverageAttr`）と**同極**の z 候補（zPeak/zRef）に差し替える。同極・中立は透過（04-magic §4.6）という通常ルールを、迂回型自身が積極的に利用する形。敵から見えるのは**前ターンまでに張られた結界**だけ（対詠では敵が先に式を晒すため、同ターンの新規結界は読めない）。

### スコアリング（`consider`）

```ts
hit = firstHit(flight, aimPos, allyHitbox)          // 失速して速度0なら候補棄却
penalty  = 手前に素材（壁）があれば 0.55（breaker は無視）
baseDmg  = hit.speed × strength × 相性 × penalty × maneuver
killBonus = baseDmg ≥ ally.hp ? 2.2 : 1            // とどめ最優先
woundFocus = 1 + (1 − hp/maxHp) × 0.5               // 手負い優先
lowHpBias  = 1 + max(0,(60 − hp)/60) × 0.25         // 低HP微優先
score = baseDmg × killBonus × woundFocus × lowHpBias
```

`maneuver` は迂回の取り回しコスト（直進=1、壁よけ=`MANEUVER=0.9`）。同条件なら直進/貫通が勝つ。

### 壁よけ軌道（`avoiderTrajectories`・#28）

breaker 以外は、直線が最初に素材へ触れる地点を迂回起点に、横へ膨らませる**通過点を選んでラグランジュ補間で近似曲線**を引く。

- 単一の弓なり：膨らみオフセット `±4, ±7, ±10` で壁の脇/上を抜ける。
- S 字：2 つの通過点で複雑に回り込む（オフセット `±6, ±9`）。機械らしい独特の軌跡。

### 牽制（フォールバック）

どの候補も命中見込みなし（命中スコア無し）なら、最も HP の低い見える味方へ**直進で牽制**（減速しない zRef で撃つ）。

### 多重詠唱（`planEnemyShots`・#44・05b §5.5）

`castCount` 本の弾を**独立に**計画して同時発射する。弾ごとに `patternPool` からパターン（role）を順繰りに選び、単一パターンの計画関数を1回ずつ呼ぶだけ（合成ロジックなし）。狙いは基本「まだ狙っていない味方」へ1発ずつ分散し、全員に行き渡ったら通常の優先度（低HP者へ集中）に戻る。

## 5.4b 崩し手の狙い（`planRuptorShot`・#42）

`ruptor` は通常の最大ダメージ探索（`consider`）を使わず、**専用の計画関数**で「誰の／どこの近くで暴発させるか」だけを決める。狙う対象の**近傍**に破綻点（z 場の極）を作る点が、他ロールと根本的に異なる。

> **原則**：`ruptor` は AI の狙い方が特殊なだけで、**物理法則・干渉ルールは味方と完全に同一**。飛行中の速度・加速度、周回リングやパリィによる迎撃、暴発そのものの挙動（04-magic §4.3）まで、既存のルールをそのまま適用する。専用の判定式・専用の免除は設けない。

### 1. 対象の選定

- **味方狙い（既定）**：見える味方の中で最もスコアの高い者（`threatScore`＝attacker の woundFocus×lowHpBias と同じ考え方）。隠蔽（§5.5）の扱いは attacker と同じ：完全に隠れた味方は狙えず、見かけ位置（`perceivedPos`）に対して計画する。
- **壁狙い（`ruptorTarget='obstacles'`・第4面デモ）**：unbreakable 以外の壁ディスクを**敵から近い順**に最大6個試し、「確実に暴発できる」狙いを選ぶ。極（g の零点）はディスク中心でなく**壁面の近傍**＝素材の外に出るまで敵側へ引いた点＋1.2 ユニット手前（中心に置くと弾が素材に削られて極に届かないため）。壁が全て崩れていれば味方狙いへフォールバック。
- **ボス断末魔（#45）**：`planEnemyShots` 経由で3本を独立計画（狙いの分散は多重詠唱の共通ルール＝3人へ1発ずつ、揃わなければ低HP者へ集中）。

### 2. 軌道と z 場の構築（`buildRuptorZField`・05b §4 が正典）

対象へ向かう軌道は、迂回型と同じ組み方（`line` を除く family＋壁よけ軌道）で通常どおり飛ばす。**破綻させるのは軌道ではなく z 場**：

```
g(x,y) = 接近方向の符号付き距離（狙点で 0。(x−tx)·ux + (y−ty)·uy）
z(x,y) = zBase×極性 + poleK'/g(x,y)      // zBase=2.2, poleK=2（RUPTOR・08-constants）
```

- `poleK'` の符号は「接近側（g<0）で極の寄与が基礎 z と同符号になる」よう選ぶ＝飛行区間は **|z|≈zBase の、属性がはっきり読める通常の光/闇弾**。
- 弾が狙点へ近づくほど `|z|` が発散し、既存の極判定（`coords.applyZValidity`：符号反転＋両側 `|z|>2·zPeak`、または非有限）がそのまま「z場のエラーによる暴発」（04-magic §4.3）として検出する。**専用の破綻フラグは設けない**。
- 候補のランク付け：「極に到達して暴発する（`pathTermination`＝invalid）」かつ「経路が壁の素材に触れない」候補を最優先し、その中で暴発点が狙点に最も近いものを選ぶ。
- 暴発点は対象のごく近傍になる。威力・範囲・付与状態異常は 04-magic §4.3 の暴発と完全に同一（**常に最大**：`power = sMax × maxFlightSpeed`、×1.5、光のひるみ＋闇の継続ダメージを両方付与）。`instability`（[04b](04b-misfire-instability.md) §4b.1）も、この暴発が**解決すれば**（＝迎撃されなければ）通常の暴発と同じく +1 される。
- 極の周辺（|g| が小さい領域）では |z|>zRef となり弾は減速する＝**着弾直前が最も遅い**。迎撃猶予はこの減速も含めて生まれる。

### 3. 予告（テレグラフ）

対詠の晒しフェーズで、他ロールと同様にゴースト軌道を見せる。加えて、**破綻点（暴発点）に警告マーカー**（赤い✕＋AoE 見込み範囲の揺れる破線円・`drawRuptureWarning`）を重ねて表示する。プレイヤーは着弾前に対応判断ができる。

### 4. 迎撃ルール（味方の弾と全く同じ物理法則に従う）

`ruptor` の弾も、**味方の弾と特別扱いを一切せず、既存のルールがそのまま適用される**。

- **属性**：破綻するまでの飛行区間は z≈zBase×極性 の通常の光/闇の弾。**極性はゴースト・弾の見た目から読み取れる**。
- **周回迎撃**：結界（周回リング）が触れたときの解決は 04-magic §4.6 の通常ルールそのまま――**反対極のみ相殺、同極・中立は透過**。ruptor の弾を止めたいなら、**その弾の属性と反対の理を帯びたリングを、破綻点へ到達する前に間に合わせる**。同極のリングでは素通りする。
- **パリィ**：味方の発射型弾との交差も 04-magic §4.5 の通常ルールそのまま――反対極のときだけ速度を削り合う。
- **速度 0 なら消滅＝暴発しない**：迎撃・相殺・壁の削りで弾の速度が 0 になれば、その場で霧散する。破綻点まで届かなければ式は評価されず、暴発は起こらない。`instability` も加算されない。
- 結界が力不足で速度を削りきれなければ、そのリングは丸ごと霧散し、弾は破綻点まで飛び続けて暴発する。
- ひるみ（光の状態異常）は他ロールと同じ：**既に放たれた飛行中の弾には影響しない**（次ターンの詠唱を阻害する効果）。

> つまり `ruptor` を防ぐ手段は、通常の迎撃・パリィと**全く同じ読み合い**――弾の属性を見て、反対の理で結界を張るかパリィで迎え撃つか。特別なメカニクスは存在しない。**迎撃に成功するかどうかは `instability` の増分に直結する。**

## 5.5 隠蔽への対応（`perceivedPos`・#35/#39）

闇の周回で隠れた味方は、敵から見て真の位置からずれて見える。

```ts
ブレ幅 mag = concealRmse（囲む円半径連動・1重=半径/2, 2重=半径）
            または重数 × CONCEAL_JITTER(=3.5)（旧データ）
見かけ位置 = 真位置 + idDir(id) × mag
```

方向は ID 由来で安定（純粋関数を保つ＝同ターンの再評価で揺らがない）。敵はこの見かけ位置に対して狙い・命中評価を行う。

## 5.6 登場する敵の一覧

各ステージの敵の完全な配置・HP・LVL・属性は [06-stages.md](06-stages.md)／設計の枠組みは [06b-difficulty-framework.md](06b-difficulty-framework.md) を参照。系統・ロールの分布だけ概観：

- **序盤（1〜2 面）**：単一系統の attacker（line / arc / wave）。
- **中盤（3〜5 面）**：火力型（breaker）・守護型（guardian）・暴発型（ruptor デモ）・高難度迂回型（slipThrough・exp/poly34・sin/cos z場）が順に登場。
- **終盤（6〜7 面）**：交互張りの守護型、崩し手3体（第6面＝初回崩壊面）、ボスは HP 380・ヒットボックス 3.6・**多重詠唱**（火力型/迂回型×2〜3本）＋HPフェーズ床崩落＋断末魔の暴発3連（#45）。

## 5.7 崩し手（ruptor）の主要パラメータ

| 定数 | 実装値 | 説明 |
|---|---|---|
| 基礎 z | `RUPTOR.zBase = 2.2` | 飛行区間の z（zRef 以下＝失速せず、属性が読める） |
| 極の強さ係数 | `RUPTOR.poleK = 2` | `z = zBase×極性 + k/g` の分子。大きいほど手前から急峻＝減速域が広がる |
| 弾速 | `castInitialSpeed = 8`（全敵固定） | 06b §2 |
| 発射頻度 | `fireEvery=2`（2ターンに1回・低頻度） | 第4面：`fireOffset=1`＝1ターン目から（デモを必ず見せる）。第6面：3体の `fireOffset` をずらして交互に撃たせる |

**残る調整項目（プレイテストで詰める）**：

- [ ] `poleK`・`zBase` と迎撃猶予のバランス（現状：極の手前 ±0.9 ユニット付近から減速）。
- [ ] 第7面ボス戦の随伴に `ruptor` を混ぜるか（現状は迂回型＋火力型の眷属のみ）。
