# 04. 魔法（二系統・暴発・パリィ・障害物・結界）

魔法の挙動を司るモジュール群：
`loop.ts`（系統分類）、`orbit.ts`（結界）、`misfire.ts`（暴発）、`parry.ts`（相殺）、`obstacle.ts`（えぐり）、`turn.ts`（解決の統合）。

---

## 4.1 二系統の分類（`src/game/loop.ts`）

軌道が「閉じている（ループ）」かで魔法を 2 系統に分ける。

```ts
classifyTrajectory(traj) = isLoop(traj) ? 'orbit' : 'projectile'
```

### `isLoop` の判定

- **回転方式 `y=g(x)`** … 常に開（`projectile`）。関数グラフは単調に外へ伸びる。
- **極座標方式 `r=f(θ)`** … 場外で切らず有限点で判定：
  1. 有限プレフィックスが 12 点未満 → 開。
  2. 始点からの最大離脱距離 `maxDepart` が 1.5 未満（ほぼ点＝退化）→ 開。
  3. 後半（点列の 50% 以降）のどこかで始点付近（許容 `max(0.6, maxDepart·0.2)`）へ戻れば → **閉（orbit）**。

円・バラ曲線・リマソンは閉じて結界に、らせんは開いて発射型になりやすい。

## 4.2 発射型（projectile・火球）

開いた軌道。術者から飛び、**最初に当たった対象**へ命中する。

解決の流れ（`turn.ts` の該当部分）：

1. `simulateFlight` で自由飛行を作る。
2. **障害物を削りながら進む**（§4.4）。途中で速度 0 なら消滅。
3. `firstHitAmong` で敵の最手前ヒットを取る。
4. guardian 敵の防御結界を横切ると減速（反対極のみ相殺・同極は透過）。削り切られると命中扱いにしない。
5. 命中なら `computeDamage` でダメージ＋状態異常。命中しなければ：
   - 軌道が `invalid` → **暴発**（§4.3）。
   - それ以外 → 外れ（miss）。

## 4.3 暴発（misfire・`src/game/misfire.ts`）

関数がエラー（未定義・発散・非実数・`1/x` の極など）になった点で魔法が綻び、**光と闇を最大強度で併せ持つ大 AoE** が起きる。

```ts
power  = sMax(=5) × maxFlightSpeed(=24) = 120       // 常に最大（速度に依らない・#41）
damage = power × AFFINITY.opposite(=1.5) = 180      // 常に有利側
AoE 半径 = aoeRadius(=5)
状態異常 = maxStatuses()                            // 光ひるみ＋闇DoT を最大強度で両方
```

- **威力は常に最大**（`sMax × maxFlightSpeed`）。暴発時の速度に依らず一定で、光・闇の両属性を最大強度で帯びる（#41）。
- AoE 内の**生存している**敵全員＋暴発点 `aoeRadius` 以内の生存味方全員にダメージ＋状態異常（撃破済み hp0 は対象外＝判定を出さない）。
- **AoE 内の壁もほぼ全て削れる**（#41）：暴発点を中心に `aoeRadius` ぶんの円を `carves` に足し、範囲内の素材を丸ごと除去する。`unbreakable`（壊れない壁）だけは残る。砕けた各 disc 位置で破片バーストが舞う。
- 暴発点が術者（原点）近傍なら**自爆**（`selfHit`）。足元で関数が壊れると味方を巻き込むので注意。
- `detectMisfire` は `pathTermination` を使い、`maxParam`（場内で完結）なら暴発しない。
- **z 場（属性関数）のエラーでも暴発する**（#30）。`sqrt(-1)`・`log(0)`・`1/x` の極など、**軌道は有効でも z 場が経路上でエラー（非有限）になる点**で同じ暴発が起きる（検出は `coords.ts` の `applyZValidity`・[03-functions.md](03-functions.md) §3.3）。暴発点はエラー手前の場内有効点。
- ステージ 6 のヒントにある通り「狙って暴発させて最大 AoE を当てる」高等戦術も成立する。

### プレビューでのエラー可視化（#30）

作成フェーズのプレビューで、関数エラーを 2 通りに可視化する：

1. **弾の暴発点（赤い ✕）**：軌道・z 場のどちらかがエラーで、弾が実際に暴発する点。`draw.ts` `drawMisfireMarker`。`computePreview` が `detectMisfire` の結果（`type==='invalid'` のみ）を `Preview.misfirePos` として返し、`App.tsx` → `BattleCanvas`（`misfirePoints` prop）→ `drawScene` の最前面に描く。場外脱出（ただの外れ）には出さない。各味方ぶん表示。
2. **場のエラー地点を全て（赤い線/領域）**：z 場を編集している間（`showZField`）、アクティブな術者の z 場が**エラーになる地点を全て**赤で重ねる（`draw.ts` `drawZFieldErrors`）。
   - **非有限**（`sqrt(-1)`・`log(0)` などの定義域外）→ 赤い**領域**。
   - **極（発散・`1/x` 型）** → 赤い**線**。場を `step=0.8` で走査し、符号反転する隣接点を二分法（`isPoleBetween`）で調べ、寄せるほど \|z\| が際限なく増大（非有限 or `ERR_BLOWUP=500` 超え）すれば極と判定する。連続関数の零点（\|z\|→0 に収束）と区別でき、急峻でも有界な場は誤検出しない。

①は「弾がどこで暴発するか」、②は「場のどこが危険か（全エラー地点）」を示す。撃つ前に式の破綻箇所を確認できる。

## 4.4 障害物のえぐり取り（`src/game/obstacle.ts`・Graph War 風）

障害物は耐久値を持たず、形は `solids`（重なった円の和＝連続ブロブ）で表す。
**魔法が当たった点を中心に円 `carves` を引き算して物理的にえぐる**。素材＝「どれかの solid 円内 ＆ どの carve 円にも入らない点」。

### 素材判定

```ts
isSolidAt(ob, p) = (どれかの solid 円内) かつ (どの carve 円にも入らない)
```

### えぐり半径（`carveRadius`）

```ts
base = min(carveMaxRadius(=2), max(0, power) × carveRadiusScale(=0.05))
半径 = base × OBSTACLE_KIND[kind].carveScale
```

威力 `power = 速度 ×(強度+1)`（中立弾でも運動量で少し掘れる）。威力が高いほど一撃で広く削れて貫通しやすい。

### えぐりの速度損（`carveSpeedLoss`）

```ts
損 = carveCost(=7) × (2 − 相性) × OBSTACLE_KIND[kind].lossScale
```

相性 1.5(反対極)→×0.5、1.0(中立)→×1、0.5(同極)→×1.5。**反対極ほど安く削れて貫通しやすい**。
速度が 0 になればその場で消滅し貫通しない（目安：最大威力で柱 1 本を約 3 発）。

### 壁の耐久種別（#40・`OBSTACLE_KIND`）

| 種別 | carveScale（半径） | lossScale（速度損） | 挙動 |
|---|---|---|---|
| `normal` | 1 | 1 | 従来の属性付き壁。light/dark で相性が効く。 |
| `fragile` | 1.7 | 0.55 | 無属性・もろい。一撃で大きく崩れる。 |
| `tough` | 0.32 | 1.7 | 無属性・頑丈。最大火力でも貫通に複数発。 |
| `unbreakable` | 0（削れない） | 1 | 壊れない。当たった魔法は**その場で全速度を失い停止**（迂回必須）。 |

実装の細部（`turn.ts` `carveAlong`）：飛行サンプルは軌道頂点なので、急な関数では頂点間が開く。`OBSTACLE_STEP=0.6` 以下に密化（`densifyGeom`）して壁のすり抜けを防ぐ。`unbreakable` に当たると全速度損で停止。

> 敵弾も同じ仕組みで障害物に削られる（障害物は味方の盾にもなる・#16）。

## 4.5 パリィ（相殺・`src/game/parry.ts`）

2 つの弾が交差したときの解決。**同極・中立はすり抜け、反対極のみ相殺**。

```ts
firstCrossing(pathA, pathB)  // 最初の交差点（線分交差）
resolveParry(attrA, speedA, powerA, attrB, speedB, powerB):
  反対極でなければ → passthrough（速度そのまま継続）
  反対極なら:
    newA = max(0, speedA − powerB × parryLossScale(=0.25))
    newB = max(0, speedB − powerA × parryLossScale(=0.25))
    0 になった側は消滅
```

発射型の味方弾が敵弾と反対極で交差すれば、互いの威力に応じて速度を削り合う。クラッシュ火花の威力は 2 魔法の威力合計（#38）。

## 4.6 軌道型＝結界（orbit・`src/game/orbit.ts`）

閉じた軌道。術者の周りを回り、**掃射（攻撃）＋迎撃（防御）＋属性オーラ（回復/隠蔽）**を兼ねる。

### リング構築（`buildRing`）

`validFinitePrefix`（場外でも切らない）で 1 周分（θ ≤ 2π）の点列＋各点の z を作る。2 周分にしないのは演出が倍速・粒子重複になるため（#22）。

### 掃射（攻撃・`orbitSweep`）

リング上で対象に最も近い点との距離が `半径 + thickness(=0.7)` 以下なら命中。

```ts
ダメージ = リング代表速度 × strengthOf(その点の z) × 相性
```

命中時、対象に属性に応じた状態異常も付与。

### 迎撃（防御・`orbitBlockLoss` / `ringInterception`）

敵弾パスがリング境界を横切ると相殺を試みる。**反対極のみ相殺、同極・中立は透過**。

```ts
削る速度 = strengthOf(リング横断点の z) × リング速度 × orbitBlockScale(=0.45)
```

- 横断点の敵弾速度 `before` に対し、`before − 削る速度 ≤ 0` なら**敵弾を止めきって結界の勝ち（存続）**。横断点で全速度を削って確実に消す。
- 止めきれなければ**結界の負け** → 一度負けたリングは**丸ごと霧散**（#34）。
- クラッシュ威力＝リング威力(強度×速度)＋敵弾威力(速度×強度)（#38）。

### 壁との接触（`orbitWallBreak`）

リングが障害物の素材に触れると、`orbitWallCarveRadius(=0.5)` でほんの少し削り、**周回は丸ごと霧散して消える**（掃射/防御/オーラを失う・#34）。

### 強属性での減速・自滅（#31/#44）

強属性（|z|>zRef）でリングは減速し、**周回速度が 0 に達すると結界も自滅して霧散する**（壁破壊と同じ霧散演出に乗せる・#31）。霧散点に小さな碎け（carve）を残し、ログに「周回結界は強すぎて失速し、自滅して消えた」と出して**自滅したことが分かる**ようにする（#44）。

そのため `zRef`(=2.5) 付近が「減速せず速く＝強いまま回り続ける」安定点になり、最強の |z|=zPeak(=5) を狙うほど早く失速して自滅するという #31 のトレードオフが成り立つ。既定の周回（z=±5）はこの自滅対象なので、結界として使い続けたいときは |z|≤zRef に乗せる。

> 自滅（失速）と「敵弾に破られて霧散」は別物。前者は敵弾なしでリングがその場で散り、後者は反対極の敵弾が貫いた点で散る（ログ文で判別できる・#44）。

### 属性オーラ（内側への効果・#35/#39）

リングが囲んだ味方に、**毎ターン**効果を及ぼす。属性は軌道上の符号付き強度の平均（`ringAverageAttr`）で決まり、効果は強度の大小を見ない固定量。入れ子は**内側（半径が小さい）優先で最大 2 つ**だけ効く。

| リング属性 | 効果 |
|---|---|
| **光** | 味方を毎ターン固定回復 `orbitHealAmount = 30`（重複可・1 重ごと） |
| **闇** | 味方を隠蔽。1 重で敵の狙いがずれ、`orbitConcealFull(=2)` 重で完全に視認不可。ブレ幅 RMSE は囲む円半径に連動（1 重=半径/2、2 重=半径） |

### 永続化（#39・`ActiveOrbit`）

一度張った結界は**破壊されるまでターンをまたいで残る**。

- 毎ターン内側へ効果（回復/隠蔽）を及ぼし、敵弾を迎撃する。
- 反対属性の弾に相殺されると消える。所有者（術者）が倒れても消える。
- 安定 ID（所有者＋リング重心＋点数）で管理し、同じ場所への張り直しは同一 ID に集約。

> guardian 敵も同じ結界の仕組みで防御リングを張る（[05-enemies.md](05-enemies.md)）。

## 4.7 解決順序（`resolveTurn`）まとめ

1 ターンの同時発射はこの順で解決される（詳細は [09-data-model.md](09-data-model.md)）：

```
1. 敵弾を構築（guardian は防御リングへ分離）
2. 味方発射を分類・構築（軌道型は壁接触で霧散。強属性で失速し速度0なら自滅して霧散・#31/#44）
3. 防御：軌道型リングの迎撃 → 発射型のパリィ（敵弾を削る・障害物も敵弾を削る）
4. 障害物：味方の発射型を削りながら遮る
5. 攻撃：命中（発射型）／掃射（軌道型）／暴発
5.5 周回の属性オーラ（回復/隠蔽）を内側の味方へ
6. 敵弾が味方へ命中
（最後に）永続周回を更新して次ターンへ持ち越し
```
