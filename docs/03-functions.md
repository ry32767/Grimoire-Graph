# 03. 関数（軌道プリセット・z 場・自由入力）

関数は 2 種類を別々に入力する：**軌道関数**（経路）と **z 場**（属性の高さ）。
`src/game/functions.ts`（軌道）、`src/game/zfields.ts`（z 場）、`src/game/mathEngine.ts`（mathjs 基盤）。

---

## 3.1 軌道プリセット（`functions.ts`）

軌道は 2 カテゴリ。各プリセットは係数スライダー（`Coefficient`）と、自由入力へコピーするための `toExpr` を持つ。

### A. 回転方式（直交 `y = g(x)`・狙う角度 θ で回転）= 発射型向き

| ID | 名前 | 式 | 係数（min〜max, step, 既定） | 説明 |
|---|---|---|---|---|
| `line` | 直線 | `y = a·x + b` | a: -6〜6, 0.1, **1** ／ b: -18〜18, 0.2, **0** | まっすぐ飛ぶ基本。狙いやすい。 |
| `parabola` | 放物線 | `y = a·(x−h)² + k` | a: -2〜2, 0.05, **0.25** ／ h: 0〜24, 0.2, **8** ／ k: -18〜18, 0.2, **-6** | 山なり/谷なりの弧。障害物を山越え。 |
| `sine` | サイン波 | `y = A·sin(B·x) + C` | A: 0〜12, 0.1, **6** ／ B: 0.1〜3, 0.05, **0.6** ／ C: -9〜9, 0.1, **0** | 波打って飛ぶ。変異が大きい。 |
| `exp` | 指数 | `y = a·e^(b·x) + c` | a: 0〜4, 0.1, **1** ／ b: -0.8〜0.8, 0.05, **0.2** ／ c: -9〜9, 0.1, **-5** | 終盤で急上昇。遠くで跳ね上がる。 |
| `abs` | 絶対値 | `y = a·|x−h| + k` | a: -5〜5, 0.1, **1** ／ h: 0〜24, 0.2, **8** ／ k: -18〜18, 0.2, **-6** | V 字に鋭く折れる。 |

### B. 極座標方式（`r = f(θ)`・全方向）= 軌道型（結界）向き

| ID | 名前 | 式 | 係数（min〜max, step, 既定） | 説明 |
|---|---|---|---|---|
| `circle` | 円 | `r = R` | R: 2〜28, 0.5, **11** | 一定半径で全周。結界の基本形。 |
| `spiral` | らせん | `r = a·θ` | a: 0.2〜4, 0.1, **1.4** | 渦巻き状に外へ。全方向を順に薙ぐ。 |
| `rose` | バラ曲線 | `r = a·cos(k·θ)` | a: 2〜24, 0.5, **12** ／ k: 2〜6, 1, **4** | 花びら状に複数方向へ。 |
| `limacon` | リマソン | `r = a + b·cos(θ)` | a: 1〜15, 0.5, **6** ／ b: 1〜15, 0.5, **6** | ハート/くぼみ形。片側に張り出す。 |

> 発射型／軌道型の判別は軌道が「閉じているか」で自動分類される（[04-magic.md](04-magic.md) §4.1）。
> 回転方式は常に発射型。極座標方式でも開いた形（らせんが一周しないなど）は発射型になりうる。

### 軌道の組み立て（`buildTrajectory`）

```ts
rotate: { mode:'rotate', g: preset.buildG(coeffs), angle, origin }
polar:  { mode:'polar',  f: preset.buildF(coeffs), origin }
```

`origin` は術者位置（発射元）。実際に撃つときは `composer.ts` の `buildComposerTrajectory` が z 場も付与する。

## 3.2 z 場プリセット（`zfields.ts`）

属性の高さ `z = f(x,y)`。符号で属性、|z| が `zPeak`(=5) に近いほど強い。V=5 を最強に合わせやすい範囲設定。

> **全員共通（#57）**：z 場は**パーティ3人で共通の単一状態**（`composer.ts` の `ZFieldState`、App の `zField` state）。キャラごとに持たず、ここを変えると全味方の弾に反映される（操作を減らすため）。UI は「属性の高さ（全員共通）」と明示し、`FunctionPanel` と盤面ボトムバーの両方が同じ `zField`／`onZChange` を編集する。
> **座標系＝ステージ中央原点（#58）**：z 場は**ステージ中央 (0,0) を原点**に絶対座標で評価する。`zfieldAt(traj,pos)` は `traj.z(pos.x, pos.y)`（origin は使わない）。**味方も敵も同じ場**を見る（敵弾も中央原点で属性が決まる）。例：`z=0.3·y` は盤面中央の横線で中立・上が光/下が闇、放射 `√(x²+y²)` は盤面中央を中心とした同心の場になる。プレビュー（`activeZField`）もずらさずそのまま描く。共通化に伴い、旧 #52 の術者位置基準は廃止。
> **係数スライダー（#52）**：z 式中の数値リテラルも軌道と同じく自動検出してスライダー化する（`zParametricPatch`／`zFitParams`）。べき指数は除外（例 `√(x²+y²)` の 2 は固定）。

| ID | 名前 | 式 | 係数（既定） | 説明 |
|---|---|---|---|---|
| `const` | 一定 | `z = c` | c: -8〜8, 0.5, **5** | どこでも同じ属性。c=+5 で最強の光、−5 で最強の闇。 |
| `gradY` | 縦勾配 | `z = a·y + b` | a: -0.6〜0.6, 0.05, **0.3** ／ b: -6〜6, 0.5, **0** | 高さ y で属性が変わる。 |
| `gradX` | 横勾配 | `z = a·x + b` | a, b 同上 | 左右 x で属性が変わる。 |
| `radial` | 放射 | `z = a·√(x²+y²) + b` | a, b 同上 | 中心からの距離で属性が変わる。 |

**狙い方**：一定 (`const`) で c=±5 にすれば全経路が最強属性。勾配 (`gradX`/`gradY`/`radial`) を使うと、「狙った位置でちょうど |z|=5 に乗せる」精密プレイができる（その分ほかの位置では弱い・速い）。

## 3.3 自由入力式（mathjs・`functions.ts` + `mathEngine.ts`）

プリセットの代わりに数式を直接入力できる。安全のため **`eval`/`new Function` は使わず mathjs のみ**。

### 変数

| 用途 | 関数 | 許可変数 |
|---|---|---|
| 回転軌道 | `parseExpression(expr, 'x')` | `x` |
| 極座標軌道 | `parseExpression(expr, 't')` | `t`（=θ） |
| z 場 | `parseZExpression(expr)` | `x`, `y`（定数式も可・errは NaN→暴発） |

加えて定数 `pi`, `e`, `tau` を許可。

### パースの安全方針（`mathEngine.ts`）

- フル mathjs ではなく**必要な関数・定数だけの限定インスタンス**を `create()` で構築（`import`/`createUnit` 等への到達を構造的に排除、バンドルも縮小）。
- **number 限定**（BigNumber/Complex/行列を排除。評価結果は常に number か例外）。
- パース木を走査し、**許可ノード以外（代入・複文・添字・オブジェクト等）と未知シンボルを静的に弾く**。
- コンパイル結果を `Map` でキャッシュ（最大 256・null も記憶して再パースを避ける）。

許可ノード：`ConstantNode`, `SymbolNode`, `OperatorNode`, `ParenthesisNode`, `FunctionNode`。

許可関数（依存注入したもの）：
`add sub mul div mod pow unaryMinus unaryPlus sin cos tan asin acos atan sqrt cbrt exp abs log log10 log2 sign min max floor ceil round hypot` ＋定数 `pi e tau`。

### エラーと暴発の分離

- **構文エラー・許可外ノード・未知変数** → `parseExpression` / `parseZExpression` は `null` を返す。UI は**直前の有効関数を維持**（壊れた式では撃たせない）。
- **式は正当だが評価が非実数/非有限/例外**（複素数・`sqrt(-1)`・`log(0)`・`1/0` の極など）→ **軌道関数・z 場ともに `NaN` を返す**（#30）。`NaN` はサンプリングで「無効点」となり、その手前の点で**暴発**として扱われる（[04-magic.md](04-magic.md) §4.3）。

この「不正式（維持）」と「正当式の発散（暴発）」の明確な分離が設計上の要点。

### 係数の自動検出と通過点フィット（`exprFit.ts`・#46）

**回転・極座標とも**、自由入力式を中心に操作する。プリセット選択時も式を自由入力欄へ転記し、係数化フローに乗せる（回転=`x`／極座標=`t`）。
通過点フィットは**射出魔法（回転 `y=g(x)`）のみ**。

**1. 係数の自動検出（数値リテラル→スライダー）**

- `mathEngine.parametrizeConstants(expr)` が式中の**数値リテラルを可変パラメータ `p0,p1,…` に置換**したテンプレートと元の値を返す。例 `2*sin(0.6*x)+1` → `p0*sin(p1*x)+p2`、`[2,0.6,1]`。
- **べき指数（`x^2` の 2 など）は係数化しない**（非整数化で複素数＝暴発になりやすいため）。
- `exprFit.detectParams(expr)` が各数値に**スライダー範囲**（値を中心に両側へ余裕・`niceStep` で刻みを整える）を割り当て `DetectedParam[]` を作る。
- スライダーを動かすと `substituteParams(template, values)` で式文字列を再生成（`freeExpr`）→プレビュー即更新。`+ -6` は `- 6` に整形（見た目のみ）。

**2. 通過点フィット（最小二乗・射出のみ）**

- フィールドをクリックして**通したい点**（数学座標）を選び、`fitToPoints` が**係数を最小二乗で最適化**して点群へ近づける。
- **局所フレーム残差**を最小化する：回転軌道は `world = origin + Rot(θ)·(x, g(x)−g(0))` なので、各点を `−θ` 回転して局所座標 `(lx, ly)` に戻すと、軌道に乗る条件は `g(lx) − g(0) = ly`。残差 `r = g(lx) − g(0) − ly`（狙い方向に直交するずれ）の二乗和を最小化する。
- **レーベンバーグ・マーカート法**（数値ヤコビアン＋減衰付き正規方程式）で決定的に解く（乱数なし＝純粋関数を保つ）。**多項式の桁違いな係数感度（`x⁴` と `x` など）にも収束する**（高次多項式を複数点へ精密フィット可能）。係数はスライダー範囲でクランプ。
- **1 点からフィット可能**（下限 1 点）。点が少なく係数が多い（劣決定）場合は LM の減衰で過剰な暴れを抑え、点を通す向きへ最小限動かす。
- **一般関数に対応（#50）**：**多スタート LM**（各係数をレンジ全体に振った初期値から最適化し最良を採る）で、sin の周波数・位相、指数、1/x（分数）など**非凸で局所解にはまりやすい関数**でも良い解へ届く。さらに**滑らかさ正則化**（局所 x の曲率＝2 階差分を小さく保つ）＋**十分近い解の中で最もうねりの少ない解を選ぶ**ことで、「点は通るが激しく振動する」過適合を避ける。
- **極（発散）回避（#50）**：1/x など分数関数で**発射経路 x∈[0,…] の前方に極がある係数は弾が暴発する**ため、`pathHasPole`（非有限／符号反転かつ両側 |g|>R_field＝±∞ を跨いだ点を検出）で**LM・選別の両方から除外**し、極を術者の後ろ側に置いて点を通す解を選ぶ。
- **打った順に通す（#50）**：射出は前方（局所 x 増加）へ掃引するため、軌道が点を訪れる順＝発射方向に沿った距離順になる。`chooseAngle` が**タップ順に単調**となる θ を選ぶ（現在の θ で既に順序通りならそのまま）。これで「打った順に通る」。`fitToPoints` は係数値と選んだ θ を返す。
- **術者位置は固定**。回転軌道は `g(x)−g(0)` を術者位置から発射するため、**加算定数（`b`/`k`/`C` など）は経路に効かない**（フィットでも動かない・既定の挙動）。傾き・高次の係数が照準・形状を決める。

### 発射方向 θ の操作（#47・射出＝回転のみ）

θ スライダーに加え、**フィールドをクリック／ドラッグして発射方向を決められる**（`App.aimAt`）。
クリック点へ向かう向きを `θ = atan2(点 − 術者)` として設定し、`[0, 2π)` に正規化する（点ピック中は無効＝点選択が優先）。
作成フェーズ中は active ally から θ 方向へ**金色の矢印**（`drawAimArrow`）を表示する。極座標には θ が無いので対象外。

> **z 場のエラーも軌道関数と同様に暴発する**（#30）。`zfieldAt` は属性・強度の評価では非有限 z を 0（中立）に丸めるが、`sampleTrajectory` が経路上で z を評価し、**非有限の点**、または**符号反転しつつ両側の \|z\| が `2·zPeak`(=10) を超える点（＝`1/x` 型の極を跨いだ）**を無効点にする（`coords.ts` の `applyZValidity`）。これにより z 場のエラー点でも発射型の弾はそこで暴発し、結界リングは途切れる。

## 3.5 おすすめ術式（`src/game/recommend.ts`）

「おすすめ」ボタンは、障害物を貫通/迂回して敵に当たる軌道＋反対極の z 場を自動探索する。

- **z 場**：敵の反対極を最強で突く（光の敵→闇 z<0、闇/中立→光 z>0）。強さ候補は `[zPeak(=5), zRef(=2.5)]` の 2 つ。
  - zPeak は近距離で大威力だが減速して失速、zRef は中遠距離でも届く。両方試して「実際に届いた威力（命中速度×強度）」が最大の方を選ぶ。
- **軌道**：`g(x) = a·x + b·x²`（直線＋わずかな放物）で、傾き候補 `a ∈ {±1,±2,±3}`、曲率 `b ∈ {0,±0.05,±0.1}` を総当たり。障害物の貫通を込みで命中速度を評価。
- 当たる候補が無ければ素直に直線（減速しない zRef）で狙う。
- 結果は直線プリセット or 自由入力式（`a*x + b*x^2`）として味方のコンポーザに反映される。
