# 04b. 暴発の不安定化・累積・崩壊（04-magic §4.3 の拡張）

> 04-magic §4.3 の暴発（misfire）を土台に、**ラン全体で累積する不安定さ**と、**三段階で開示される崩壊**（隠蔽 → 初回崩壊＝グリモワールの介入で救済 → 致死）を定義する。テーマ：**滅亡の原因は暴発の濫用**であることを、説明ではなく**プレイ体験と段階的な気づき**で示唆する。
> 実装：`src/game/misfireInstability.ts`（純粋関数）＋ `src/game/turn.ts`（`ResolveResult.misfires` / 半径ばらつきの適用）＋ `src/App.tsx`（ラン状態・イベント・UI）。定数は `src/data/constants.ts` の `INSTABILITY` / [08-constants.md](08-constants.md)。既存の暴発（power 常時最大・AoE 基準半径 `aoeRadius=5`・光闇両属性）はそのまま。

---

## 4b.1 不安定カウンター（instability・味方＋敵の合算・ラン全体で累積）

「膜の摩耗」を表すカウンター。**ステージ・ターンをまたいでラン全体（全7面通し）で持ち越す**。

```
instability : 整数、0 から上へ
+1 ： 暴発が「解決」するたび（味方でも敵でも同じ +1、意図的でも事故でも）
     ※ プレビュー段階（未確定）はカウントしない。実際に暴発した回数だけ。
     ※ 実装：resolveTurn が返す misfires（{pos, owner}[]）の件数を App が加算する。
```

- **味方も敵も同じ重み**。滅亡の“主犯”は特定の誰かではなく、**盤上で起きた暴発すべて**が膜を削る（＝都市の滅びは濫用そのものが原因、という含み）。敵にも暴発を撃つ個体（崩し手）を置き、その暴発も +1 する。
- **減らない**（濫用のツケが残る）。緩和として、**暴発ゼロでステージクリア**時に `−1`（`applyStageClearRelief`・下限0・実装済み）。
- **リトライ**：ゲームオーバーでステージをやり直すと、そのステージ**開始時点の値へ巻き戻す**（ラン全体では持ち越し）。
- **世界観**：撃つほど平面と高次を隔てる膜がすり減る（[lore.md](lore.md) §5）。

## 4b.2 三段階の開示（隠蔽 → 初回崩壊 → 致死）

**カウンターは最初は見せない。** プレイヤーは異変と暴発円のブレだけで危うさを感じ、初回崩壊で正体を知り、以後はルールが明示される。

### 第1幕 ― 隠蔽期（カウンター非表示）

- **数値・メーターは一切出さない。**「あと何回」は分からない。
- 伝える手がかりは2つ：
  1. **ステージの異変**（`anomalyLevel`・`draw.ts drawAnomaly`）：背景が波打ち、床にひびが走り、崩壊目前は画面周縁が赤黒く脈動する。※駆動音（音の異変）は未実装。
  2. **暴発円のブレ**（4b.3）：暴発プレビューの✕の周囲に min–max の二重リングが出る。
- 異変の段階（`anomalyLevel(count)`）：

| instability | 段階 | 異変 |
|---|---|---|
| 0–1 | 0 | なし（静か） |
| 2–3 | 1 | 背景がわずかに波打つ |
| 4 | 2 | 床に明確なひび |
| 5 以上 | 3 | ひび増加＋画面周縁が赤黒く脈動 |

### 転機 ― 初回崩壊（グリモワールの介入・一度だけ）

`instability` が `firstCollapseThreshold(=5)` に達したとき、**または遅くとも第 `firstCollapseStage(=6)` 面のターン2以降で必ず**、初回崩壊が起きる（`shouldFirstCollapse`・閾値優先・指定面で保証）。第6面の崩し手3体（§4b.4b）が、この閾値を確実に越えさせる。

1. 演出：物語オーバーレイ（`COLLAPSE_FIRST`・[story.md](story.md)）＝暴発の連鎖 → 暗転 → **グリモワールに関係する“何か”**が結界で食い止める。
2. 救済の後、はじめて**膜メーターが可視化**され（`Hud.tsx InstabilityMeter`）、以後「**あと何回で崩壊するか**（`misfireLimit − instability`）」が常時表示される。

- この救済は**一度きり**。カウンターはリセットしない（“何か”は overflow を受け止めただけ）。以後は救われない。
- ここは伏線でもある：グリモワール（第七層でようやく辿り着く書物）が、**すでに、どこかから見ていて、手を伸ばせる**こと。ソウが感じ続けた「上でも下でもない方角からの視線」の正体を、断定せず匂わせる。

### 第2幕以降 ― 致死期（メーター可視・崩壊＝ゲームオーバー）

- メーターと「あと何回」を常時表示（残り2回以下で赤く点滅）。
- `instability` が `misfireLimit(=12)` に達する暴発は制御を失い、**ステージ全体を覆う暴発 → ゲームオーバー**（`isLethal`・救済なし）。勝敗判定より**先**に評価する＝**ボスを倒した直後でも破局しうる**（免除なし・[06b-difficulty-framework.md](06b-difficulty-framework.md) §6 第7面）。
- **確認ゲート**：メーター開示後、その発射のプレビューに暴発が含まれ、それが解決すると上限に達しうる場合、発射ボタンが一度 **「⚠ 崩壊の危険 ― それでも発射」** に変わり、もう一度押したときだけ発射する（式を変更するとゲートは解除）。
- 破局演出：専用のゲームオーバー文（`GAMEOVER_COLLAPSE`）＝中心から放射状に光と闇が噴き出し盤面を呑む（第七層「放射状の焦げ跡」と同じ絵＝**都市の最期の追体験**）。

## 4b.3 不安定化＝半径のばらつき（プレビュー円がブレる）

`instability` が上がるほど、暴発 AoE 半径に**ばらつき**を与える。**平均は変えず分散だけ広げる**（第1幕から作用＝隠蔽期の主要な手がかり）。

```
v(count) = clamp((count − vStart(=2)) / (misfireLimit − vStart), 0, 1) × vMax(=0.6)
実半径   = clamp(aoeRadius × (1 + U(−v, +v)), 1, ∞)   // 発射解決時に確定（misfireRadius）
```

- **乱数の注入**：ロジックは純粋関数のまま、`resolveTurn` に `misfireRoll ∈ [0,1)` を渡す（App が `Math.random()`、テストは固定値）。同一ターン内の2発目以降は黄金比ステップでロールを進める。`roll=0.5` でちょうど `aoeRadius`（ブレなし）。
- **プレビュー**：暴発点の✕の周りに `[aoeRadius×(1−v), aoeRadius×(1+v)]` の**二重リング**（`drawMisfireBand`）。危険は見えるが、正確な大きさは読み切れない。
- 実半径はダメージ判定・壁のえぐり（`misfireCarveWalls`）の両方に適用される。※爆発**演出**の見た目半径は基準半径のまま（既知の未対応・要改善）。
- 威力・両属性・状態異常は従来どおり（半径だけが不安定）。半径が広がれば自爆リスクも増す。

## 4b.4 敵暴発によるステージ崩壊イベント（選択式・分岐／致死期）【未実装・将来テンプレート】

**メーター開示後**の局面に、敵が暴発を溜めて**ステージが崩壊しかける**「溜め式・阻止可能」イベントを置く案。**第七層ボスの最終仕様（06b §6 第7面）では、HP0時の「断末魔の暴発3連」が確定発生する形に置き換えられたため、本イベントは現行7面では不採用＝未実装**。将来の追加ステージや高難度モードのテンプレートとして仕様だけ残す。

- 専用ゲージ `stageCollapse`：崩し手（敵）が暴発詠唱を開始し、`collapseTurns(=2)` で満ちる（毎ターン可視化）。この敵暴発も解決すれば `instability +1`。
- **阻止条件**：満ちる前に、反対の理での相殺／大ダメージ、または**ひるみ（光）**での詠唱阻害。
- **分岐**：期限内に阻止＝崩壊回避（任意で `instability −1`）／満了＝部分崩壊（大 AoE・地形消失・即敗北ではない）／最悪＝全崩壊 → ゲームオーバー。

## 4b.4b 敵ロール「崩し手（ruptor）」と暴発の学習導線

> AI の詳細（狙いの選定・z場の極による暴発の起こし方・迎撃ルール・パラメータ）は [05-enemies.md](05-enemies.md) §5.3・§5.4b・§5.7 と [05b-enemy-archetypes.md](05b-enemy-archetypes.md) §4 を正典とする。ここではカウンタ／ステージ演出との接続だけを扱う。

暴発を撃つロール **`ruptor`（崩し手）**（05-enemies §5.3）で、プレイヤーに暴発を **提示 → 誘発 → 崩壊で痛感** の順で学ばせる導線を作る。

### ロール `ruptor`（狙い撃ち型）

- **味方（または第4面デモでは障害物）を狙い、z 場に極を仕込んで対象近傍で暴発**（最大 AoE・光闇両属性・×1.5・両状態異常＝04-magic §4.3）を起こす。自分の足元ではなく**対象付近**で暴発させる（自爆はしない）。
- **予告あり**：対詠の晒しフェーズで、ゴースト軌道に加えて**暴発予告（赤い✕＋不安定に揺れる円）**を表示（`drawRuptureWarning`）。
- **着弾前に反対の理で防げる**：崩し手の弾も味方の弾と同じ物理法則・干渉ルールに従う（特例なし）。**極到達の前に、弾と反対極の結界（周回リング）またはパリィで速度を0にすれば、暴発しない**（04-magic §4.6/§4.5 の通常ルールそのまま。同極のリングは素通りする）。防げた暴発は AoE を起こさず、`instability` も積まない。※ ひるみ／詠唱阻害は他の敵弾同様、飛行中の弾には効かない。
- **防がなければ確実に積む**：迎撃を怠った暴発は必ず解決して `instability +1`。

### 学習導線（ステージ配置・[06b-difficulty-framework.md](06b-difficulty-framework.md) §6）

| 面 | 崩し手 | ねらい |
|---|---|---|
| **第4面** | **1 体**（低頻度 `fireEvery=2`・1ターン目から・**岩壁狙い** `ruptorTarget='obstacles'`） | 敵の暴発を初めて見せ、「**これは自分たちも撃てる技だ**」と気づかせる（提示・RUPTOR_DEMO）。初の敵暴発時に一度だけオーバーレイで「関数をわざと壊すと暴発する／撃つほど膜は摩耗する／反対極の結界・パリィで防げる」を補足（`CODEX_MISFIRE_HINT`） |
| **第6面** | **3 体**（`fireEvery=2`・位相をずらして交互に撃つ・味方狙い） | 盤面で暴発が多発。合算カウンターが一気に伸び、**第6面で初回崩壊（グリモワール救済）が戦闘中に発生**。「濫用すると崩れる」を体で覚えさせる（誘発→崩壊） |

- **第6面の3体は反対極の結界・パリィで個別に防げる**（着弾前に速度0で暴発せず・積まない）。だが3体が異なる属性・タイミングで撃ち続けるため、結界の張り替えが追いつかなければ `instability` は急伸する。仮に全て防ぎ切っても、**初回崩壊は第6面（ターン2以降）で必ず発生**する（§4b.2 の保証面＝ストーリー上の一度きりの演出）。つまり崩壊の“発生”は保証しつつ、プレイヤーの結界運用が**その後の残り枠（メーター）**を左右する。

- [x] 初回崩壊は「暴発→暗転→グリモワールに関係する“何か”が結界で食い止める」テキスト演出で、**カウンターはリセットせず**、以後メーター（あと何回）を常時表示する

## 4b.7 ナラティブ接続（[lore.md](lore.md) / [story.md](story.md)）

- **膜メーターの上昇 ＝ 都市が辿った道**。味方も敵も暴発が膜を削る＝「濫用そのもの」が滅びの原因という含み。
- **初回崩壊のグリモワール介入**：正体を明かさぬまま“何か”が救う＝グリモワールが**すでに関与している**伏線。ソウの「視線」の回収を**匂わせるが断定しない**。
- **第六層の初回崩壊／致死期の破局／第七層の断末魔3連**：滅びを**追体験・再現**し、真相（[lore.md](lore.md) §5「何か＝都市規模の暴発」）を**明言せず**突きつける。
