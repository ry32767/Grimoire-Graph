# Graph Mage（グラフメイジ）

> 関数を「描いて」魔法に変える、ブラウザで動くレトロRPG。失われた古代遺跡を舞台に、敵の関数を読んでから自分の関数で対抗する、ターン制の関数バトル。1プレイ約15分で完結します。

古代文明は世界の理を「数式」で記しました。それが魔法文字＝術式です。プレイヤーは遺跡を巡る魔導士見習いとして、古代式を読み解いて魔法を放ちます。プリセットの関数ボタンを選び、スライダーで形を整えるだけで「術式の軌道」を作れるので、数式に苦手意識があっても遊べます。各ターンは、まず**敵が関数を見せ、それを見たうえでこちらの関数を組み、同時に発射**して決着します。さらに `z = f(x, y)` の「理の場」の上で、弾の当たった位置が正なら《光の理》・負なら《闇の理》になり、`z` が 0 に近いほど弾が加速し、`速度 × |z|` で威力が決まります。中立帯で加速して強属性帯に差し込む——そんな駆け引きが楽しめます。サーバーやデータベースは不要で、ブラウザだけで完全にローカル動作します。

## 主な機能

- **ターン制の読み合い** — 各ターン、敵が先に関数を見せる。それを読んでから自分の関数を組み、両者が同時に発射して決着する
- **関数で狙う軌道づくり** — プリセット関数 + 係数スライダー + 自由入力で曲線を調整。狙う角度を回す方式と、極座標 `r=f(θ)` で全方向に展開する方式の両方が使える
- **数学が苦手でも遊べる補助** — 関数の意味を一言で説明し、`f(0)=…` など計算結果の一部を数値で表示。発射前に予測着弾点・属性・威力の目安を見せ、危ない式には暴発警告。確定するまで何度でも試せて、困ったらおすすめ関数をワンタップ
- **光と闇の極性属性** — `z = f(x, y)` の「場」の上で、弾を当てた位置が正なら光・負なら闇。0 から離れるほど強い。場は背景に薄く色分け表示
- **加速度と運動量の物理** — `z` が 0 に近いほど弾が加速。運動量を保ったまま飛び、障害物やパリィで速度が減り、速度が 0 になると消滅する
- **威力は速度 × 属性の強さ** — 中立帯で加速して、当てる瞬間に強属性帯へ差し込むほど威力が出る
- **円で防御（シールド）** — `x²+y²=1` のような閉曲線を張って敵弾の速度を削り、止める
- **障害物を避ける／壊す** — 属性付きの障害物が道をふさぐ。避けるか、相性のいい属性で壊して道を開ける
- **暴発とパリィ** — 関数がエラーになった地点で魔法が暴発。暴発は光と闇を最大強度で併せ持つ切り札になる一方、`1/x` のような式は足元で暴発して自爆するリスクも。敵弾とぶつかったときは、同じ属性同士ならすり抜け、光と闇の反対属性同士なら相殺し合う
- **約15分のライトストーリー** — タイトル → 数ステージ → エンディングまで一気に遊べる

> 各機能の詳細な受け入れ条件（"完成"の基準）は [graph_mage-spec.md](graph_mage-spec.md) を参照。

## 世界観とデザイン

- **テーマ**: 失われた古代文明の遺跡を巡る魔導士見習いの物語。数式は古代の魔法文字＝術式。遺跡には《光の理》と《闇の理》が満ちる。
- **トーン**: 8ビット・ドット絵調、ノスタルジック。石碑や魔導書を思わせるチャンキーな枠、ピクセル表現。
- **配色（高コントラスト・背景黒め）**: 光の理＝金〜白、闇の理＝紫〜藍、背景はほぼ黒。理の場は `f>0` を金、`f<0` を紫藍、中立を暗いグレーで色分け。

> 配色・フォント・トーンの詳細は [graph_mage-spec.md](graph_mage-spec.md) の §3.9 を参照。

## 技術スタック

- **フロントエンド**: React 18 + TypeScript + Vite
- **描画**: HTML5 Canvas（バトルフィールドと関数グラフ）
- **数式パース**: mathjs（自由入力された式の安全な評価）
- **バックエンド / DB**: なし（完全ローカル動作）
- **デプロイ**: ローカル（`npm run dev`）／ 静的ホスティング（GitHub Pages 等）も可

## セットアップ

```bash
# 依存をインストール
npm install

# 開発サーバ起動（ブラウザで http://localhost:5173 を開く）
npm run dev

# 本番ビルド（dist/ を生成）
npm run build

# テスト / 静的解析
npm test          # Vitest（ゲームロジックの純粋関数）
npm run lint      # ESLint
npm run typecheck # tsc --noEmit
```

`main` ブランチへの push で GitHub Actions（`.github/workflows/deploy.yml`）がテスト → ビルド → GitHub Pages へ自動デプロイします。

## 使い方

1. タイトル画面で「はじめる」を押す。
2. 戦闘画面で、プリセット関数（直線・放物線・サインなど）を選ぶ。
3. スライダーで関数の形を整え、曲線が敵に当たるように狙う。属性表示を見て敵の弱点を突く。
4. 「発射」を押すと弾が曲線に沿って飛ぶ。敵に当たればダメージ。
5. 全ての敵を倒すとステージクリア。数ステージ進めてエンディングへ。
