# Graph Mage（グラフメイジ）

> 関数を「描いて」魔法に変える、ブラウザで動くレトロRPG。失われた古代遺跡を舞台に、敵の関数を読んでから自分の関数で対抗する、ターン制の関数バトル。1プレイ約15分で完結します。

古代文明は世界の理を「数式」で記しました。それが魔法文字＝術式です。プレイヤーは**3人の術者からなる自陣営**を率い、遺跡を巡って敵チームと戦います。プリセットの関数ボタンを選び、スライダーで形を整えるだけで「術式の軌道」を作れるので、数式に苦手意識があっても遊べます。各ターンは、まず**敵が関数を見せ、それを読んだうえで味方3人それぞれの関数を組み、全員＋敵が同時に発射**して決着します。属性は**自分が描いた関数の値そのもの**（高さ `z`）で決まり、関数値が正なら《光の理》・負なら《闇の理》、`z` が 0 に近いほど弾が加速し、`速度 × |z|` で威力が決まります。中立帯で加速して当てる瞬間に強く帯びさせる——そんな駆け引きが楽しめます。サーバーやデータベースは不要で、ブラウザだけで完全にローカル動作します。

## 主な機能

- **3人パーティ vs 敵チーム** — 自陣営の術者3人それぞれに関数を割り当て、敵チームと同時に撃ち合う。味方タブで切り替え、「全員発射」で一斉解決
- **ターン制の読み合い** — 各ターン、敵が先に曲がった術式を見せる。敵の頭上の記号〔直進/弧/波/渦〕で攻撃の形を読み、回避・迎撃する
- **関数＝属性（3D化）** — 属性は描いた関数の値そのもの。関数値が正なら光・負なら闇、0 から離れるほど強い。当てる位置の関数値を大きくするほど高威力（`速度 × |z|`）
- **2系統の魔法** — 開いた形＝**発射型**（火球）、閉じた形＝**軌道型**（円など）。軌道型は術者の周りを回り、敵弾を弾く防御も兼ねる（＝結界）
- **固定初速の物理** — 初速は毎回固定。`z` が 0 に近いほど弾が加速し、障害物やパリィで減速、速度 0 で消滅する。演出の速さは弾の実速度に一致
- **障害物を削って貫通** — 属性付きの障害物は当てるほど半径が削れ、削り切れば貫通。途中で速度が 0 になれば弾は消える
- **暴発とパリィ** — 関数がエラー（`1/x` など）になった直前の地点で魔法が暴発（光と闇の大 AoE）。足元なら自爆注意。敵弾とは反対の理同士で相殺（パリィ）
- **敵AIと得意関数** — 敵は系統ごとの得意関数を持ち、最大ダメージ・とどめを狙って狙い角と形を最適化してくる
- **音と演出** — Web Audio 合成の効果音と BGM（ミュート可）、ドット爆発・多層グローの魔法演出
- **チュートリアル＋5ステージ＋ボス** — クリアタイムを計測表示。約15分で一気に遊べる

> 各機能の詳細な受け入れ条件（"完成"の基準）は [graph_mage-spec.md](graph_mage-spec.md) を参照。

## 世界観とデザイン

- **テーマ**: 失われた古代文明の遺跡を巡る魔導士見習いの物語。数式は古代の魔法文字＝術式。遺跡には《光の理》と《闇の理》が満ちる。
- **トーン**: 8ビット・ドット絵調、ノスタルジック。石碑や魔導書を思わせるチャンキーな枠、ピクセル表現。
- **配色（高コントラスト・背景黒め）**: 光の理＝金〜白、闇の理＝紫〜藍、背景はほぼ黒。弾の軌道は関数値 `z` で色分け（正＝金、負＝紫藍、中立＝淡色）。

> 配色・フォント・トーンの詳細は [graph_mage-spec.md](graph_mage-spec.md) の §3.9 を参照。

## 技術スタック

- **フロントエンド**: React 18 + TypeScript + Vite
- **描画**: HTML5 Canvas（バトルフィールドと関数グラフ）
- **数式パース**: mathjs（自由入力された式の安全な評価）
- **バックエンド / DB**: なし（完全ローカル動作）
- **デプロイ**: ローカル（`npm run dev`）／ 静的ホスティング（GitHub Pages 等）も可

## セットアップ

```bash
# 依存をインストール
npm install

# 開発サーバ起動（ブラウザで http://localhost:5173 を開く）
npm run dev

# 本番ビルド（dist/ を生成）
npm run build

# テスト / 静的解析
npm test          # Vitest（ゲームロジックの純粋関数）
npm run lint      # ESLint
npm run typecheck # tsc --noEmit
```

`main` ブランチへの push で GitHub Actions（`.github/workflows/deploy.yml`）がテスト → ビルド → GitHub Pages へ自動デプロイします。

## 使い方

1. タイトル画面で「はじめる」を押す。
2. 戦闘画面で味方タブを切り替え、3人それぞれにプリセット関数（直線・放物線・サインなど）や自由入力式を割り当てる。
3. スライダーと角度で形を整え、曲線が敵に当たるように狙う。属性表示と敵の系統記号を見て弱点を突く。円（閉曲線）を描けば周回結界として防御も兼ねる。
4. 「全員発射」を押すと味方3人と敵が同時に解決。敵に当たればダメージ、暴発や迎撃も発生。
5. 全ての敵を倒すとステージクリア。チュートリアル＋5ステージ＋ボスを進めてエンディングへ（クリアタイムも表示）。
